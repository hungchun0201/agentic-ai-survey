S1

工作負載特徵分析

Agentic LLM 工作負載在五個關鍵維度上與傳統 chatbot 流量根本不同：多輪深度（每個程式 50–200+ 次推理呼叫 vs 1–3 輪）、工具呼叫引起的暫停碎片化 GPU 利用率、單調增長的 context 壓迫 KV cache 容量、跨輪次和跨 agent 的冗餘 prefill、以及CPU 主導的延遲（工具執行時間遠超 LLM 推理時間）。理解這些模式是基礎設施共同設計的先決條件。

Continuum — KV Cache TTL for Agent Scheduling
arXiv'252511.02230repo Agentic ✓PD —
提供最詳細的 agent 流量實證分析，使用 SWE-Bench 和 BFCL trace。發現大多數 tool call 在 <2 秒內完成且高度可預測；scheduling bubbles 累積達 baseline 端到端延遲的 58.2%。開源了首個公開 agentic inference benchmark trace。
The Cost of Dynamic Reasoning
HPCA'262506.04301 Agentic ✓PD —
首次全面的系統級分析，比較 ReAct、Reflexion、LATS、LLMCompiler 四種 agent 框架。揭示「迫在眉睫的永續性危機」：agent 雖以增加計算換取精確度，但出現急劇遞減的回報、不斷擴大的延遲變異、以及數量級的成本增長。
A CPU-Centric Perspective on Agentic AI
arXiv'252511.00739repo Agentic ✓PD —
剖析五種代表性工作負載（Haystack RAG、Toolformer、ChemCrow、Langchain、SWE-Agent），揭示被忽略的瓶頸：CPU 上的工具處理可消耗高達 90.6% 的總延遲。提出 CGAM 和 MAWS，在 Intel Emerald Rapids + NVIDIA B200 上實現 2.1× P50 延遲降低。
PrefillShare — Shared Prefill for Multi-LLM
arXiv'262602.12029 Agentic ~PD Disagg
量化多模型 agent 管線（Planner→Coder→Reviewer）中的冗餘 prefill，提出跨模型共享 prefill 模組以實現 KV 重用。
AgentInfer — Co-Design of Inference Architecture
arXiv'252512.18337 Agentic ✓PD —
將 agent 延遲分解為 LLM 呼叫、工具呼叫、檢索和環境互動，提出推理架構與系統的共同設計框架。
S2

Prefill-Decode Disaggregation

Prefill-decode (PD) disaggregation——將計算密集的 prefill 階段和記憶體頻寬密集的 decode 階段分配到不同 GPU 池——已成為 2025 年的主流 serving 架構。然而，agentic 工作負載以其極高的 KV cache 重用率（≥95%）將瓶頸從計算轉移到 KV cache I/O，需要架構重新思考。

DistServe — PD Disaggregation for Goodput
OSDI'242401.09670repo Agentic —PD DisaggDistributed
奠定 PD disaggregation 基礎，獨立優化各階段的資源分配和並行策略。7.4× 吞吐量或 12.6× 更緊的 SLO。到 2025 年幾乎所有生產級系統都採用了此架構。限制：FCFS 排程、無多輪 KV cache 重用。
Splitwise — Phase Splitting for LLM Inference
ISCA'242311.18677repoBest Paper Nominee Agentic —PD DisaggDistributed
描繪各階段的硬體特性：prefill 為計算瓶頸、decode 為記憶體頻寬瓶頸，功耗特性各異。實現異質硬體部署（H100 prefill + A100 decode），1.4× 吞吐量、低 20% 成本。
Mooncake — KVCache-centric Architecture
FAST'252407.00079repo★ Best Paper Agentic ~PD DisaggDistributed
Kimi（Moonshot AI）的生產級 serving 平台，每日處理 1000 億+ tokens。利用未充分使用的 CPU/DRAM/SSD 構建分散式 KV cache pool，Transfer Engine 在 4×200 Gbps RoCE 上達 87 GB/s。已整合至 vLLM、SGLang、TensorRT-LLM。
DualPath — Storage Bandwidth Bottleneck Fix
arXiv'262602.21548 Agentic ✓PD DisaggDistributed
最直接相關的 agentic disaggregation 論文。發現在 KV cache 命中率 ≥95% 時，prefill 端的 storage NIC 飽和而 decode 端閒置。提出 storage→decode→prefill 的新 RDMA 路徑，1.87× 離線吞吐量、1.96× 線上吞吐量。挑戰了 DistServe 時代的計算中心優化觀點。
TaiChi — Hybrid Aggregation-Disaggregation
arXiv'252508.01989 Agentic —PD DisaggDistributed
嚴格證明純聚合或純分離都非普遍最優。提出 P-heavy/D-heavy GPU instance 和「延遲轉移」策略——選擇性降低已滿足 SLO 請求的延遲以幫助高風險請求，77% goodput 提升。
NVIDIA Dynamo — Smart Router + KV Manager

最具生產就緒性的 PD disaggregation 框架，具備明確的 agentic 功能：radix-tree KV cache 追蹤的 Smart Router、SLA-based 動態 P/D scaling 的 Planner、分層 offloading 的 KV Block Manager。v1.0 新增 cache pinning TTL（nvext.cache_control）和 LangChain 整合。GB300 NVL72 上 30× 吞吐量。
TraCT — CXL Shared Memory KV Cache
arXiv'252512.18194 Agentic —PD DisaggDistributed
使用 CXL（Compute Express Link）共享記憶體作為無網路的 KV 傳輸基底，透過直接 load/store 操作實現機櫃級 KV cache 共享。TTFT 降低 9.8×、P99 延遲 6.2×。CXL Type-3 設備仍處預生產階段。

ISCA'25 Agentic —PD DisaggDistributed
基於 stream 的 disaggregation，在 GPU instance 內實現非同步並行執行。動態 prefill dispatch 和無停頓重排程。4.28× TTFT 中位數、1.5× TPOT P99。
NanoFlow — Nano-grained Pipeline Parallelism

透過 nano-grained pipeline parallelism 實現接近最優吞吐量。OSDI'25 Best Paper。

S3

KV Cache 管理與 Prefix Caching

KV cache 管理是對 agentic 工作負載影響最大的單一優化。該領域已從基本的分頁記憶體管理快速發展到分散式、位置無關、工作流感知的快取系統。

vLLM — PagedAttention
SOSP'232309.06180repo Agentic —PD —
引入 OS 啟發的虛擬記憶體和分頁機制用於 KV cache，將 tensor 分割為固定大小 block 並使用 copy-on-write 共享。近零記憶體浪費、2–4× 吞吐量提升。奠定所有後續 KV cache 系統的基礎。限制：單 GPU、無跨請求重用、基本 FCFS eviction。
SGLang — RadixAttention
NeurIPS'242312.07104repo Agentic ~PD —
以 radix tree 儲存 KV cache 實現跨 LLM 呼叫的自動重用，支援 few-shot、多輪、tree-of-thought 和 agent 模板模式。Cache-aware scheduling 優先處理較長共享前綴的請求，達 5× 吞吐量。默認 LRU eviction 對 agentic 次優（未來激活可預測）。
LMCache — Multi-tier KV Storage Layer
arXiv'252510.09665repo Agentic ~PD DisaggDistributed
首個開源 KV caching layer，支援跨查詢前綴重用和 PD disaggregation。跨 GPU HBM、CPU DRAM、磁碟、遠端儲存（Redis、Mooncake），達 15× 吞吐量。Pin API 提供 agent session 的顯式保留控制。已整合至 vLLM 生產級堆疊。
CachedAttention — Hierarchical KV Persistence
ATC'242403.19708 Agentic ~PD —
為多輪對話實作分層 KV cache 持久化（DRAM + 磁碟），具備位置編碼解耦和層級預載入。在 ShareGPT trace 上達 86–90% cache 命中率。
llm-d — K8s-native KV Cache Indexer
OSS'26repo Agentic ~PD DisaggDistributed
Red Hat/IBM/Google 的 Kubernetes 原生分散式推理。全域 KV-Cache Indexer 的 Precise Prefix-Cache Scorer，87.4% cache 命中率、88% 更快 TTFT。Session affinity 防止多輪 agent 的 cache thrashing。v0.5 新增 cache-aware LoRA routing。
CacheGen — KV Cache Compression Codec
SIGCOMM'242310.07240repo Agentic —PD —Distributed
使用客製化 codec 將 KV cache tensor 編碼為緊湊 bitstream，利用分佈特性實現 3.5–4.3× 壓縮、TTFT 降低 3.2–3.7×。分散式節點間 agent context 傳輸的關鍵技術。
CacheBlend — Non-prefix KV Cache Reuse
EuroSys'252405.16444repo★ Best Paper Agentic —PD —
實現非前綴文本 chunk 的預計算 KV cache 重用，透過選擇性重計算 5–18% 邊界 token 恢復跨 chunk 注意力。2.2–3.3× TTFT、2.8–5× 吞吐量。直接適用於動態組裝多文件 context 的 RAG agent。
EPIC — Position-Independent Caching
ICML'252410.15332repo Agentic —PD —
以 compile-link 類比形式化 Position-Independent Caching：chunk 被「編譯」（獨立 KV cache）再「連結」（邊界選擇性重計算）。8× TTFT、7× 吞吐量。對文件排序在不同調用間變化的 agent 特別有價值。

SOSP'252501.12689 Agentic —PD —Distributed
語義級快取，利用歷史請求-回應對作為 in-context example 增強較小模型。兩階段 example 選擇和成本感知路由，1.4–5.9× 吞吐量、28–71% 延遲降低。利用 70%+ 的 agent 查詢有語義相似歷史對應的觀察。
KVFlow — Workflow-aware Prefix Caching

Marconi — Prefix Caching for Hybrid LLMs

S4

KV Cache TTL 與保留策略

Cache 保留——決定在工具呼叫暫停期間在 GPU 記憶體中保留 KV cache 多久——可以說是 agentic 工作負載最高槓桿的優化，但仍然是研究最少的領域。核心矛盾：保留 cache 避免昂貴的 re-prefill 但阻塞 GPU 記憶體給其他請求；過早驅逐浪費 prefill 投資。

未解問題：（1）無自適應 TTL——目前值在請求時設定且從不更新；（2）無跨程式交互建模；（3）無語義感知保留；（4）無分散式 TTL 協調；（5）無原則性的 exploration-exploitation 框架。
InferCept — KV Cache Preserve for Tool Calls
ICML'242402.01869repo Agentic ✓PD —
引入更簡單的「preserve」操作，在 tool call 之間釘選 KV cache。後續工作（Continuum、KVFlow、ThunderAgent）指出關鍵限制：靜態 pin 策略在高變異性工具呼叫時失敗——長時間或失敗的呼叫無限期阻塞 GPU 記憶體。
Concur — AIMD Agent Admission Control
arXiv'262601.22705 Agentic ✓PD —
辨識 batch agentic inference 中的「middle-phase thrashing」：長壽命 agent 在並發壓力下 cache 效率崩潰。提出受網路壅塞控制啟發的 AIMD-based agent 准入控制，在高並發下優於 Continuum。
ThunderAgent — Program-aware Pause/Restore
arXiv'262602.13692repo Agentic ✓PD —
Program-aware inference，透過遲滯水位線（hysteresis watermarks）進行週期性記憶體監控和 pause/restore 操作。當 KV cache thrashing 即將發生時，主動暫停活躍 agent。在可預測 tool-call 時間的工作流上優於 vLLM 和 Continuum 2.43–3.56×。
SideQuest — Model-driven KV Eviction
arXiv'262602.22603 Agentic ✓PD —
採取根本不同的方法：使用推理模型本身透過輔助「side quest」任務決定驅逐哪些 KV entry。解決了多步驟推理中 token 重要性非單調的根本問題。降低深度研究 agent 的峰值 token 使用 65%。
S5

排程與路由

傳統 LLM 請求排程將每次推理呼叫獨立處理。Agentic 工作負載需要程式級和工作流級排程，理解多步驟依賴、利用 cache 局部性、適應動態執行圖。

Autellix — Program-level DAG Scheduling
arXiv'252502.13965 Agentic ✓PD —Distributed
首個將 agent 程式視為一等公民的 serving 系統（UC Berkeley & Google DeepMind）。將程式形式化為動態 DAG，引入 PLAS 排程（優先服務累計服務時間較低的程式）和 ATLAS（多執行緒版本，考慮關鍵路徑）。Locality-aware 負載均衡器利用 KV cache co-location，4–15× 吞吐量。
Cortex — Stage-isolated Resource Pooling
arXiv'252510.14126 Agentic ✓PD —Distributed
Stage isolation——為每個工作流階段（如 SQL 生成、錯誤修復、執行）配置獨立資源池，各有自己的排程器、cache 和 scaling 策略。Orchestrator 追蹤 per-request SLO slack。
Murakkab — Declarative Workflow Optimization
arXiv'252508.18298 Agentic ✓PD —Distributed
Microsoft Research 提出宣告式抽象，將工作流規格與執行配置解耦。Profile-guided 優化器和自適應 runtime 聯合管理模型選擇和硬體映射，2.8× GPU 使用量降低 vs LangGraph。
Agent.xpu — On-device SoC Scheduling
arXiv'252506.24045 Agentic ✓PD Disagg
首個面向異質 SoC（CPU、iGPU、NPU）的 on-device agentic 引擎。Prefill 分配到 NPU、decode 分配到 iGPU。4.6× 更低延遲（reactive）、1.6–6.8× 更高吞吐量（proactive）。Intel Core Ultra 平台。
Preble — Cluster-level KV-aware Scheduling
ICLR'252407.00023repo Agentic —PD —Distributed
將 prefix caching 擴展到叢集級，E2 排程演算法共同優化 KV 重用和負載均衡。全域排程器的 prompt-aware 負載指標考量驅逐成本和共享潛力，1.5–14.5× 平均延遲、2–10× P99。
AI Metropolis — Out-of-order Multi-Agent
MLSys'252411.03519 Agentic ✓PD —
將 out-of-order 執行概念應用於多 agent 模擬，動態追蹤 agent 間依賴以打破保守同步屏障。1.3–4.15× 加速。
Helium — Query-plan Agentic Workflows
arXiv'262603.16104 Agentic ✓PD —
將 agentic 工作流建模為資料庫查詢計劃，LLM 作為一等運算元，搭配 template-aware radix-tree 排程。
Sherlock — Reliable Agentic Workflow
arXiv'252511.00330 Agentic ✓PD —
結合選擇性驗證與投機執行，用於可靠的 agentic serving。執行時間降低 48.7%。
Aragog — Just-in-Time Model Routing
arXiv'252511.20975 Agentic ✓PD —
在執行期間按輸入自適應（而非按工作流），實現 Just-in-Time 模型路由以擴展 agentic 工作流。
SOLA — State-aware SLO Scheduling
MLSys'25 Agentic —PD —
State-aware SLO 排程用於 LLM serving。MLSys'25。

S6

學習式快取管理

最優 KV cache TTL 和驅逐本質上是 exploration-exploitation 問題：系統必須在利用已知存取模式與探索保留不確定 cache entry 的價值之間平衡。豐富的學習式快取文獻提供了理論基礎。

LeCaR — Regret-based LRU/LFU Weighting
HotStorage'18repo Agentic —PD —
引入 regret-minimization 動態加權 LRU 和 LFU。每次 cache miss 時根據累積 regret 選擇策略。在 cache 相對工作集較小時，優於 ARC >18×。CACHEUS 的基礎。
CACHEUS — Fully Adaptive Cache Replacement
FAST'21repo Agentic —PD —
擴展 LeCaR 至完全自適應，透過 gradient-based stochastic hill climbing 消除所有靜態超參數。引入工作負載分類和 SR-LRU/CR-LFU 新 expert。在 329 工作負載上的 17,766 次模擬中評估。
RLCache — Multi-task RL Cache Management
arXiv'191909.13839repo Agentic —PD —
將快取管理建模為多任務 RL 問題，含三個 agent：快取決策、TTL 估計、驅逐。TTL 估計 agent 學習最優 time-to-live——直接相關於 agentic semantic caching。
Cold-RL — Offline RL for NGINX Eviction
arXiv'252508.12485repo Agentic —PD —
首個生產級 RL 驅逐演算法，整合至 NGINX。Dueling DQN 透過 ONNX sidecar 在嚴格 500µs timeout 內運作。命中率從 0.14 提升到 0.35（146% 提升），CPU 開銷 <2%。
