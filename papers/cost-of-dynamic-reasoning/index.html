The Cost of Dynamic Reasoning: Demystifying AI Agents and Test-Time Scaling from an AI Infrastructure Perspective

HPCA 2026

Jiin Kim, Byeongjun Shin, Jinha Chung, Minsoo Rhu

KAIST

1 Overview & Motivation

Figure 1 (from paper): Overview of reasoning strategies in LLM-based systems. (a) Conventional LLMs map inputs directly to outputs. (b) Reasoning-enhanced LLMs create intermediate steps. (c) AI agents augment reasoning by planning, invoking external tools, observing outcomes, and iteratively refining decisions.

Deploying reasoning-enhanced LLMs comes at immense computational cost. Even static reasoning models run on thousands of GPUs costing tens of millions of dollars monthly. AI agents with dynamic reasoning—which actively interact with external environments through multi-step tool use—threaten to exacerbate these pressures by orders of magnitude.

This paper presents the first comprehensive system-level characterization of AI agents, quantifying resource utilization, latency behavior, energy consumption, and datacenter-scale power demands across diverse agent workflows and benchmarks. The key finding: a looming sustainability crisis where agents exhibit rapidly diminishing returns, widening latency variance, and unsustainable infrastructure costs.

Central Thesis
AI agent performance does not scale proportionally with compute, energy, and power costs. Once accuracy saturates, additional test-time scaling yields diminishing returns while imposing substantial system-level burdens. This cost inefficiency poses concrete constraints on real-world deployments.

Latency Breakdown

On average, LLM inference accounts for 69.4% and tool execution 30.2% of total latency. These two phases alternate sequentially, creating a fundamental bottleneck: the LLM output is needed to determine which tool to invoke, and the tool result feeds back into the next LLM call.

LLMCompiler Exception
LLMCompiler attempts to mitigate this sequential dependency through asynchronous tool execution during planning. The observed LLM+Tool overlap accounts for only 18.2% of total latency, showing partial but limited success.

Latency Distribution: Chatbot vs. Agent

A conventional chatbot (ShareGPT) produces responses with a tight latency distribution, with 95% of requests completing within 9.7 seconds. In contrast, a ReAct agent on HotpotQA exhibits a much broader distribution with a heavy tail, where the 95th percentile can extend to 20-60+ seconds due to variable reasoning steps and tool interactions.

Token Usage Patterns

Compared to CoT, AI agents have significantly longer inputs because their contexts include accumulated LLM history and tool history tokens. For HotpotQA, initial inputs of ~1,000 tokens grow to 3-4x as prior outputs and tool responses are appended. This creates substantial prefix overlap across consecutive LLM calls—an opportunity for prefix caching.

Impact of Prefix Caching

Compute Savings

Prefix caching reduces prefill latency by an average of 60.1% for agent workloads. In LATS, where parallel LLM calls share prefixes, memory savings reach 64.8%.

Memory Impact

Tool-augmented agents consume 3.0x more KV cache memory per request than CoT (up to 5.4x worst case) due to iteratively growing contexts.

Agent vs. Chatbot Throughput

While ShareGPT (conventional chatbot) sustains up to 6.4 QPS, ReAct agents achieve only 2.6 QPS on HotpotQA and 1.2 QPS on WebShop—even with prefix caching. The multi-step reasoning with repeated LLM invocations fundamentally limits throughput.

Effect of Prefix Caching on Serving

At the serving level, prefix caching achieves a 5.62x average throughput increase for ReAct by avoiding redundant prefill operations across requests. For ShareGPT (single-turn), the improvement is a mere 1.03x. Prefix caching also reduces average and maximum KV cache memory by 51.7% and 63.5% respectively.

Iteration Budget Effects

Increasing the maximum iteration budget for ReAct on HotpotQA reveals a critical pattern:

  • Both accuracy and average latency eventually saturate
  • The 95th percentile latency continues to increase linearly, driven by outlier tasks consuming full iterations
  • These outliers degrade cost-efficiency without yielding substantial accuracy gains
  • The widening latency distribution is especially problematic for latency-sensitive deployments

Sequential vs. Parallel Scaling

Sequential Scaling (Reflexion)

More reflection steps improve accuracy with diminishing returns. Going from 16.9s to 25.6s yields a 4% accuracy gain; achieving the same improvement from 56.0s requires an additional 269.5s—a 31x higher cost for the same marginal gain.

Parallel Scaling (LATS)

Increasing child nodes from 1 to 16 improves accuracy by 14.4 percentage points while reducing latency by 196.3s on average. Exploring multiple paths in parallel helps converge on answers faster, but at the cost of increased memory pressure from concurrent LLM requests.

Model Size Effects

Both Llama-3.1-Instruct 8B and 70B eventually reach accuracy saturation. The 70B model achieves high accuracy with relatively low latency, while the 8B model requires more reasoning iterations. However, the 8B model is substantially more energy-efficient: running on a single GPU versus eight, it achieves lower total energy consumption per request even when requiring more reasoning steps.

Compensatory Scaling
The performance gap between models can be partially closed with effective scaling strategies. With LATS and parallel scaling, the 8B model achieves near-70B performance by exploring multiple paths—demonstrating that test-time strategies can compensate for smaller model capacity.

7 Implications & Future Directions

Key System-Level Challenges

Serialized Control Flow

The sequential LLM↔Tool dependency limits intra-request parallelism. GPU idle periods can reach 54.5% of total execution. Pipelining and prefill-decode disaggregation are essential.

KV Cache Pressure

Iterative context accumulation causes 3-5.4x more KV cache memory than single-turn. Long-context agents with growing histories exacerbate GPU memory contention.

Unpredictable Latency

Variable reasoning depths produce heavy-tailed latency distributions, making SLA management extremely challenging for production deployments.

Diminishing Returns

Once accuracy saturates, additional test-time scaling yields marginal gains while costs grow linearly or super-linearly. The cost-efficiency frontier is essential for sustainable deployment.

// Optimization landscape for sustainable agent deployment

class SustainableAgentDesign {
  // 1. Model-level optimizations
  static model_opts = [
    "quantization",       // INT4/INT8 reduce memory + compute
    "distillation",       // smaller models for sub-tasks
    "sparse_architectures",// MoE for selective activation
    "adaptive_routing",   // route tasks to right-sized model
  ];

  // 2. System-level optimizations
  static system_opts = [
    "prefix_caching",       // 5.62x throughput for agents
    "kv_cache_compression", // reduce memory footprint
    "speculative_decoding", // exploit schema patterns
    "prefill_disaggregation",// separate compute/memory phases
  ];

  // 3. Agent-level optimizations
  static agent_opts = [
    "iteration_budgeting",  // cap reasoning depth
    "carbon_aware_compute", // migrate to low-carbon regions
    "adaptive_scaling",    // adjust compute by task difficulty
    "hybrid_model_routing", // SLM for planning, LLM for acting
  ];

  static optimalConfig(task, latency_budget, power_budget) {
    return paretoFrontier(
      accuracy: task.evaluate(),
      cost: measureLatency() + measureEnergy(),
      constraints: { latency_budget, power_budget }
    );
  }
}

Promising Directions

  1. Heterogeneous multi-agent systems: combining SLMs and LLMs, dynamically selecting the appropriate model based on sub-task significance (planning vs. acting)
  2. Carbon-aware computing: migrating latency-insensitive agent tasks to lower-carbon-intensity compute regions
  3. Adaptive scaling strategies: dynamically adjusting compute resources based on task difficulty to avoid over-provisioning
  4. Non-prefix KV cache reuse: extending beyond prefix caching to handle the complex sharing patterns in agent contexts
  5. Prefill-decode disaggregation: decoupling compute-intensive prefill from memory-bound decode for flexible resource allocation

8 Key Takeaways

The Looming Sustainability Crisis

AI agent performance does not scale proportionally with its associated compute, energy, and power costs. The paper calls for a paradigm shift from unconstrained test-time scaling toward compute-efficient reasoning that balances performance with deployability under real-world constraints.

Orders-of-Magnitude Cost Increase

Agent workloads consume 62-137x more GPU energy per query than conventional single-turn inference. This gap is fundamental to the multi-step reasoning paradigm.

Diminishing Returns at Scale

Accuracy saturates while latency and energy grow linearly or worse. The 95th percentile latency widens continuously, degrading predictability for production systems.

Parallel Scaling Advantages

LATS shows parallel exploration can reduce latency while improving accuracy—but at the cost of increased memory pressure from concurrent LLM requests.

Prefix Caching is Essential

With 5.62x throughput improvement and 60.1% prefill reduction, prefix caching is a critical optimization for agent serving, far more impactful than for static LLMs.

Small Models + Smart Scaling

8B models with parallel scaling (LATS) can achieve near-70B performance with lower total energy, suggesting heterogeneous deployment strategies.

Gigawatt-Scale Infrastructure

At search-engine-scale traffic, agentic workloads demand power rivaling national grids—making infrastructure sustainability a first-class design constraint.
