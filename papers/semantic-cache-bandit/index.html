Semantic Caching for Low-Cost LLM Serving

The first principled, learning-based framework for semantic cache eviction under unknown query distributions — from offline optimization to online adaptation with provable guarantees.

Liu, Atalar, Dai, Zuo, Wang, Chen, Joe-Wong · UW, CMU, CUHK, CityU HK, Microsoft Research

Overview

Semantic caching reduces LLM inference costs by reusing previously computed responses for semantically similar queries. Unlike exact-match caching (string/token matching), semantic caches reason about meaning: "What is LLM caching?" and "How does caching work in large language models?" should share a cached response despite differing surface forms. Over 30% of LLM queries exhibit such semantic similarity, yet existing systems rely on ad-hoc eviction policies with no theoretical foundations.

This paper introduces a principled framework grounded in combinatorial multi-armed bandits that addresses three progressively harder settings: oracle (known distributions), offline (learned from data), and online adaptive (learned in real-time with low switching costs).

System Model

Semantic Caching System Architecture

Formal Definitions

  • Queries Q = {q_1, ..., q_m}: Finite set of distinct queries, each a natural language prompt.
  • Embeddings e: X → R^{d_e}: Maps queries to vectors (e.g., via OpenAI Embedding API).
  • Arrival probability p(q): Unknown probability that query q arrives at each round.
  • Serving cost c(q) ∈ (0,1]: Unknown expected cost of querying the LLM for q.
  • Distance d(q1, q2): Semantic similarity metric (Euclidean distance between embeddings).
  • Cache M ⊆ Q, |M| ≤ k: Set of at most k query-response pairs stored in a vector DB.

Decision Rule & Loss Function

a_t = LLM(q_t) if c(q_t) ≤ d(q_t, M)   |   a_t = cached response otherwise

The expected loss of a cache M balances mismatch cost against serving cost:

l(M; p, c, d) = Σ_{q ∈ Q} p(q) · min{c(q), d(q, M)}
Key Properties (Lemma 2)

The loss function is non-increasing (caching more never hurts) and supermodular (diminishing returns from each additional cached item). These properties enable greedy algorithms with provable approximation guarantees.

Oracle Setting: Reverse Greedy

When p and c are fully known, the problem is still NP-hard (reduces to maximum vertex cover on bipartite graphs). The authors design Algorithm 1:

// Algorithm 1: Reverse Greedy for Oracle Setting
Input: Queries Q, probabilities p, costs c, distance d, cache size k
M_0 ← Q                        // Start with ALL queries cached

for i = 1 to m - k do
    // Find least beneficial cached query
    for each q in M_{i-1}:
        compute l(M_{i-1} \ {q}; p, c, d)
    q_i ← argmin_q l(M_{i-1} \ {q})  // least loss increase
    M_i ← M_{i-1} \ {q_i}               // remove it
end for
return M = M_{m-k}                    // final cache of size k
Theorem 1 (Approximation Guarantee)

Let M be the Reverse Greedy solution and M* the optimal cache. Then: l(M; p, c, d) ≤ (e^β - 1) / β · l(M*; p, c, d) where β = c/(1-c) and c ∈ [0,1] measures how far the loss deviates from additive (curvature). For the exact-match special case, this recovers optimal guarantees.

Offline Setting: CUCB-SC

When p and c are unknown but a historical dataset D = {(M_t, q_t, C_t)}_{t=1}^n is available, the paper introduces CUCB-SC (Combinatorial UCB for Semantic Caching).

// Algorithm 2: CUCB-SC (Offline Learning)
Input: Dataset D, queries Q, embedding e, distance d, cache size k, δ

for each q in Q do
    p_hat(q) = N(q) / n                       // empirical arrival probability
    c_hat(q) = mean(costs for q)             // empirical cost
    c_bar(q) = c_hat(q) + sqrt(log(6mn/δ) / (2*N_c(q)))
                                                // UCB: pessimistic cost estimate
end for

M_hat = Reverse_Greedy(Q, p_hat, c_bar, d, k)
return M_hat

Key Design Decisions

U

Upper Confidence Bound (UCB) Costs

Costs are estimated pessimistically by adding the confidence interval. This follows the pessimism principle: with limited data, overestimate costs to make robust cache decisions.

E

Empirical Arrival Probabilities

Full-feedback structure: every query arrival is observed (unlike costs which are only observed when the LLM is queried). This allows direct use of empirical means for p.

G

Greedy Oracle Integration

The estimated parameters feed into the Reverse Greedy oracle, inheriting its approximation guarantees with added statistical error terms.

Theorem 2 (Suboptimality Gap)

With probability at least 1-δ, the suboptimality gap of CUCB-SC scales as O(sqrt((Σ 1/ν(q)) · log(mn/δ) / n)) where ν(q) is the probability of observing cost feedback for query q. The gap decreases at rate sqrt(1/n) with the number of offline data samples.

Online Setting: CLCB-SC-LS

The most challenging setting: the agent interacts with a live stream of users, simultaneously serving queries, learning unknown distributions, and updating the cache — all while minimizing the number of expensive cache switches.

CLCB-SC-LS Online Learning Loop

Three Key Mechanisms

  1. Stage-Based Cache Switching: The cache is only updated when sufficient new observations accumulate, controlled by a threshold that doubles geometrically. This bounds total switches to O(m log log T).
  2. Lower Confidence Bounds (LCBs): Costs are estimated optimistically (LCBs instead of UCBs) to encourage querying the LLM for uncertain queries, collecting valuable cost feedback.
  3. Serve and Update: Between cache switches, the agent serves queries using the current cache while continuously refining its parameter estimates from observed feedback.
Theorem 3 (Regret Bound)

The regret of CLCB-SC-LS over T rounds is bounded as: Reg(T) ≤ O(sqrt(mT log(mT)) · log log T). The number of cache switches is bounded by O(m log log T), providing up to 85% reduction in runtime compared to baselines that switch every round.

Special Case: Exact-Match Caching

When the distance function uses a threshold (d=0 if embeddings are within ε, d=1 otherwise), the framework reduces to traditional exact-match caching. In this degenerate case, the offline result improves upon prior work (Zhu et al.) by a factor of O(k/sqrt(C_1)) where C_1 is the minimum query cost, thanks to the separate treatment of arrival and cost distributions.

Experimental Results

Experiments use a synthetic dataset with m=20 queries generated by ChatGPT with varying semantic similarity, 384-dimensional sentence-transformer embeddings, and Euclidean distance normalized to [0,1].

Oracle Setting Results

Reverse Greedy vs. Brute Force

Reverse Greedy perfectly approximates the brute force optimal loss for all cache sizes k=3..10, validating Theorem 1. LFU (Least Frequently Used) is consistently worse.

Optimal Loss vs. Cache Size

Loss decreases supermodularly with cache size, confirming Lemma 2. The marginal benefit of caching one more query diminishes as expected.

Offline Setting Results

With m=20 and k=5, CUCB-SC achieves the lowest final suboptimality gap across stream lengths 300-600. The LCB variant (CLCB-SC) and Epsilon-Greedy perform worse, confirming that pessimistic cost estimation is crucial in the offline setting.

Relevance to Agentic AI Systems

1

Multi-Agent Query Caching

In multi-agent architectures where agents issue similar queries (e.g., multiple coding agents hitting the same API documentation), semantic caching dramatically reduces redundant LLM calls. The bandit framework learns which shared queries to cache.

2

Cost-Aware Serving

Agentic systems face heterogeneous costs: some queries require expensive reasoning models, others need cheap lookups. The mismatch cost framework naturally balances quality degradation against computational savings.

3

Adaptive Cache Management

As agent workloads shift over time (e.g., from coding to research to writing), CLCB-SC-LS adapts the cache online with provably low switching costs, avoiding the expense of rebuilding caches from scratch.

4

MCP Server Integration

Tool-calling agents using MCP servers can cache tool responses (API results, database queries) using semantic similarity. The framework's distance-function generality supports arbitrary similarity metrics for different tool types.

Bridge: Bandit Theory Meets LLM Serving

This work provides the first rigorous connection between multi-armed bandit theory and semantic LLM caching. For agentic AI infrastructure, it offers a principled alternative to ad-hoc caching policies like LRU or LFU, with provable regret bounds that guarantee near-optimal cost reduction even under completely unknown workload distributions.

Loss Function Deep Dive

Supermodularity and Greedy Guarantees

The loss function l(M; p, c, d) has two critical structural properties that enable efficient approximate optimization:

N

Non-Increasing (Lemma 2i)

For any A ⊆ B ⊆ Q and q ∉ B: l(B; p, c, d) ≤ l(A; p, c, d). Adding more query-response pairs to the cache never increases the expected loss. More cached items means more queries can be served from cache.

S

Supermodular (Lemma 2ii)

l(A ∪ {q}) - l(A) ≤ l(B ∪ {q}) - l(B). The marginal benefit of caching one more query diminishes as the cache grows. This is the diminishing returns property that enables greedy approximation.

Concentration Inequalities

The theoretical guarantees rely on careful concentration arguments for both arrival probabilities and serving costs:

// Lemma 3: Concentration of Estimates (high probability)
// For any delta > 0, define events:

E_arvl := { |p_hat(q) - p(q)| ≤ sqrt(2m*log(2/δ) / n)  for all q }
E_cost := { |c_hat(q) - c(q)| ≤ sqrt(log(2mn/δ) / (2*N_c(q)))  for all q }
E_counter := { N_c(q) ≥ n*p(q)*ν(q) / 2  for all q }

// All three hold with probability at least 1-delta when:
n ≥ 8*log(m/δ) / min_q(p(q)*ν(q))

Sensitivity Analysis (Lemma 4)

The loss function sensitivity to parameter perturbations is bounded:

|l(M; c, p, d) - l(M; c', p', d)| ≤ Σ_q |p(q) - p'(q)| + Σ_q p(q)|c(q) - c'(q)|

This Lipschitz-like property ensures that small estimation errors in p and c lead to proportionally small errors in the loss, enabling finite-sample guarantees for the offline and online algorithms.

Algorithm Design Insights

Why Reverse Greedy (not Forward Greedy)?

Standard forward greedy algorithms build up the cache by iteratively adding the most beneficial query. Reverse Greedy starts with all queries cached and iteratively removes the least beneficial one. For minimizing a non-increasing supermodular function under a cardinality constraint, the reverse greedy approach provides tighter approximation guarantees (via Corollary 4 of Ilev 2001).

Why LCBs in Online (not UCBs)?

In the offline setting, CUCB-SC uses upper confidence bounds (pessimistic costs) because with limited historical data, overestimating costs leads to conservative but safe cache decisions. In the online setting, CLCB-SC-LS uses lower confidence bounds (optimistic costs) to encourage exploration: underestimating a query's cost makes the agent more likely to query the LLM for it, collecting valuable cost feedback that improves future estimates.

Exploration-Exploitation in Caching

The online caching problem uniquely combines two challenges: (1) the agent only observes serving costs when it queries the LLM (partial feedback), and (2) changing the cache is expensive (switching costs). CLCB-SC-LS addresses both by using optimistic cost estimates to drive exploration and stage-based switching to limit cache updates. The result is sublinear regret O(sqrt(mT) log log T) with only O(m log log T) cache switches.

Ablation Studies

Cache Size k

Both CLCB-SC and CLCB-SC-LS achieve lower final regret as k increases, while CUCB-SC and LFU see higher regret. The low-switching algorithms better exploit larger caches.

Number of Queries m

CLCB-SC-LS and CLCB-SC achieve lower final regret than baselines across all values of m, validating the tight regret bound in Theorem 3.

Switching Overhead

CLCB-SC-LS reduces switches by up to 90.91% and runtime by up to 85.40% compared to Epsilon-Greedy, while maintaining comparable or better regret performance.

Future Directions

  • Hybrid Offline-Online: Integrate pre-deployment offline training with online adaptation for faster convergence in production deployments.
  • Non-Stationary Distributions: Extend to settings where query arrival probabilities and costs change over time (concept drift).
  • Hierarchical Caching: Multi-tier caches (GPU memory, CPU RAM, SSD) with different access costs and capacity constraints.
  • Personalized Caching: Per-user or per-session cache policies that adapt to individual usage patterns in multi-tenant deployments.
  • Content-Aware Eviction: Incorporate query content features (beyond embeddings) for more nuanced similarity assessment.
  • Integration with Speculative Serving: Combine semantic caching with speculative tool calling for compound latency savings in agentic workloads.
