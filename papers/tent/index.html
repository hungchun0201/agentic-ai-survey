3.3 Where the Logic Lives

Orchestrator (heavy, ~31k LOC)

  • Topology discovery & tier assignment
  • Plan synthesis (direct + multi-hop)
  • Slice scheduler with telemetry
  • Failure detection & recovery
↓

Backends (thin, <800 LOC each)

  • Submit a single slice to the fabric
  • Report completion + latency
  • Nothing else — no retry, no routing
Why this split matters: The split keeps fabric-specific code tiny and scheduling policy uniform.

5. Phase 2 — Telemetry-Driven Slice Spraying

5.1 Why Slice, and Why 64 KB

64 KB balances doorbell overhead against tail-latency variance.

5.2 The Scoring Model

Linear model over queue depth and bandwidth:

(1) $$\hat{t}_a \;=\; \beta_{0,a} \;+\; \beta_{1,a} \cdot \frac{A_a + L}{B_a}$$

Coefficients updated via EWMA; topology penalty creates soft priority.

(2) $$s_a \;=\; P_{\text{tier}}(d) \cdot \hat{t}_a, \qquad a^{\star} = \arg\min_{a \in \mathcal{N}(L,R)} s_a$$

Tolerance window prevents pathological stickiness.

5.3 Feedback Loop

1. Submit slice with predicted $\hat{t}_a$

↓

2. Record actual completion time $t_a$

↓

3. EWMA update

↓

4. Periodic reset (30 s) avoids starvation

5.4 Multi-Tenant Extension

Optional global queue-depth blending for multi-tenant NIC pools.

6. Phase 3 — Proactive Dual-Layer Resilience

6.1 Link-Level Recovery

Detect

  • Implicit: EWMA error spikes
  • Explicit: RDMA/CUDA errors, timeouts
↓

Exclude (soft)

  • Set $P_{\text{tier}} \to \infty$
  • Reassign outstanding slices
↓

Probe + re-admit

  • Heartbeat slices (1 s)
  • Gradually restore penalty

Absolute offsets make slice retries idempotent.

6.2 Transport-Level Failover

Backend-level failover demotes transport one tier down.

7. Phase 4 — Low-Overhead Datapath

Per-slice bookkeeping must stay cheap.

Lock-free MPSC rings

Single atomic append per submission.

Pinned per-context workers

No doorbell contention.

One-sided receive

Zero-copy, zero-wakeup landings.

Hierarchical atomic counters

Sharded counters keep atomics cold.

11. Takeaways & Open Questions

Strengths

  • Decoupled intent/execution.
  • Simple cost model.
  • Idempotent slices.
  • Battle-tested at production scale.

Limitations & Open Questions

  • Shared-uplink accuracy unclear.
  • No public GB200 NVL72 decode study.
  • Joint scheduling with KV layer.
  • Reset period is hard-coded.

11.1 Position in the Agentic Infrastructure Stack

Transfer engine becomes load-bearing in agentic workloads.

Bottom line: The operational story matters more than the numbers.
