#1 Overview

The First Prefix Caching System for Hybrid LLMs

Marconi is the first prefix caching system designed for hybrid Large Language Models that combine Attention layers with State Space Models (SSMs) such as Mamba. While prefix caching is standard practice in Transformer-only serving systems (SGLang, vLLM), hybrid architectures introduce a fundamental new challenge: SSM layers maintain recurrent hidden states that depend on the entire input history, preventing the simple rolling-back of partial prefix overlaps that KV-cache reuse relies on.

Marconi solves this with novel admission and eviction policies that forecast reuse likelihood of cached entries and weigh compute savings against memory footprints, yielding up to 34.4x higher token hit rates than existing prefix caching approaches.

Why Prefix Caching Breaks for Hybrid Architectures

In a pure Transformer, the KV cache at position i depends only on tokens 1 through i, and each position's KV pair is independent. Reusing a prefix means simply copying the KV blocks and continuing decoding from the divergence point. Hybrid models with SSM layers break this assumption in two critical ways:

  • In-place state updates: SSM recurrent states (ht) are updated in-place as each token is processed. There is no per-token KV pair; only the final aggregate state after processing L tokens.
  • No partial rollback: When a new request shares a prefix but diverges mid-sequence, there is no way to "roll back" the SSM state to the divergence point unless it was explicitly checkpointed.
  • Asymmetric memory costs: Each SSM state checkpoint costs a fixed amount regardless of sequence length (D x N x 2 bytes), while KV cache costs scale linearly with sequence length.

#4 Admission & Eviction Policies

V1: SGLang+ (Baseline)

  • LRU eviction on leaf nodes only
  • All ancestor timestamps updated on access
  • No awareness of compute savings vs memory cost
  • Admits all prefixes indiscriminately
  • Single signal: recency of access

V2: Marconi

  • Utility-based eviction on leaf + single-child nodes
  • Only the deepest accessed node's timestamp is updated
  • Weighs FLOPs savings / memory footprint per node
  • Selective admission via radix tree structure
  • Two signals: recency + compute efficiency, balanced by alpha
Utility(node) = α · efficiency(node) + recency(node)
Where efficiency = FLOPs_saved / memory_footprint (min-max normalized), recency = 1 / (now - last_access) (min-max normalized), and α is tuned retrospectively via grid search.

Eviction Policy V2 -- Compute-Aware Node Scoring

Marconi's eviction policy (implemented in radix_cache_hybrid.py:evict_v2()) collects all nodes with 0 or 1 children, then computes a utility score for each:

  1. FLOPs Efficiency Calculation: For each candidate node, compute the FLOPs saved by caching it (Mamba FLOPs + Attention FLOPs + MLP FLOPs) divided by its memory footprint (SSM states + KV cache).
  2. Recency Score: Inverse of time since last access, with only the deepest node's timestamp updated on lookup -- not ancestors.
  3. Combined Utility: Weighted combination: α * normalized_efficiency + normalized_recency
  4. Eviction: The node with the lowest utility score is evicted first. Leaf nodes are deleted; single-child nodes are merged with their child.
Critical Difference from SGLang: SGLang (V1) updates the timestamps of all ancestor nodes on every access, making parent nodes appear artificially "hot." Marconi (V2) updates only the deepest accessed node, allowing cold intermediate nodes to be evicted and merged with their children -- freeing SSM state memory.

#5 Retrospective Alpha Tuning

Core Data Structures

class HybridStates:
    """Encapsulates the heterogeneous cached states for a radix tree node."""
    def __init__(self, input_ids, mamba_states, kv_cache):
        self.input_ids = input_ids      # original tokens for this node
        self.mamba_states = mamba_states  # SSM recurrent + conv states (all layers)
        self.kv_cache = kv_cache          # Attention KV pairs (per-layer, per-token)

class TreeNode:
    """A node in the radix tree, storing a segment of token IDs and cached states."""
    def __init__(self, logical_ts=None):
        self.key = None           # tuple of token IDs (parent -> this node)
        self.value = None         # list of token IDs
        self.parent = None        # parent pointer
        self.children = defaultdict(TreeNode)  # keyed by first token ID
        self.last_access_time = logical_ts   # for LRU / utility scoring
        self.hybrid_states = None             # HybridStates object

#8 Agentic AI Relevance

SWE-Agent / SWEBench Evaluation

Marconi is explicitly evaluated on SWE-Agent workloads derived from the SWEBench Verified dataset. This is a code-editing agentic benchmark where an LLM agent iteratively reads code, executes commands, and modifies files over many turns to resolve GitHub issues.

SWE-Agent sessions exhibit the highest cache pressure among all evaluated workloads because:

  • Each turn appends the full conversation history (system prompt + all prior observations and actions)
  • Sessions can span 50+ turns, with context lengths reaching 32K tokens
  • The environment observations are collapsed (except the last 5) following SWE-Agent guidelines to manage context length
  • Cache sizes range from 40-100 GB, reflecting the massive state requirements

Why Agentic Workloads Benefit Most: Agentic LLM applications (code agents, tool-using agents, multi-step reasoning) generate long, heavily-overlapping request sequences with growing context windows. This is precisely the regime where Marconi's compute-aware eviction outperforms naive LRU, because the cost of re-computing SSM states for long prefixes is enormous while the memory savings from evicting them are modest.

#9 Integration with SGLang

SGLang as the Foundation

Marconi builds directly on top of SGLang's radix-tree-based prefix caching architecture. The paper frames SGLang's existing approach as "V1" (SGLang+) and Marconi as "V2" -- an enhanced replacement for SGLang's eviction policy module.

Key integration points:

  • Same radix tree structure: Marconi uses the same tree node layout as SGLang, adding HybridStates (Mamba states + KV cache) at each node
  • Drop-in eviction replacement: The evict_policy_version flag switches between SGLang's LRU (V1) and Marconi's utility-based eviction (V2)
  • Selective admission: Where vLLM caches all token blocks, Marconi/SGLang+ only cache at radix tree branching points -- naturally filtering for shared prefixes
  • Timestamp semantics: The critical behavioral change: V2 only updates the deepest accessed node's timestamp, vs V1 updating all ancestor timestamps

Important Observation -- Simulation vs Runtime: The open-source implementation is a trace-driven simulator, not a GPU runtime integration. Actual tensor states (Mamba hidden states, KV cache tensors) are not materialized; instead, the code tracks node metadata (token IDs, timestamps, session IDs) and computes memory/FLOPs costs analytically using the formulas in utils.py. This is the standard approach for cache policy research -- the TTFT results from the paper were measured separately on a real serving system (data in data/ttft_AI21-Jamba-1.5-Mini.pickle).

Code Quality and Reproducibility Notes

  • Artifact evaluation: Full reproduction instructions provided in artifact_evaluation.md with conda environment, automated dataset download via gdown, and end-to-end experiment scripts.
  • Logical timestamps: The code uses use_logical_ts=True by default for exact reproducibility -- eliminating non-determinism from wall-clock timestamps.
  • Bootstrap multiplier tuning: LMSys uses a higher multiplier (15x vs 5x default) because its sessions have more turns and the cache fills more slowly, requiring a longer observation window for reliable alpha estimation.
  • Minor code note: A comment on line 89 of radix_cache_hybrid.py flags self.num_nodes as potentially buggy. In practice, the counter is correctly incremented/decremented in insert/delete/merge operations and is used only for pretty-printing, not for algorithmic decisions.
