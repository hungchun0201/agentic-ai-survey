CACHEUS extends the LeCaR framework from a clever proof of concept to a fully adaptive, production-ready cache replacement algorithm. It eliminates all static hyperparameters through three key innovations: an adaptive learning rate via gradient-based stochastic hill climbing, scan-resistant and churn-resistant expert policies (SR-LRU and CR-LFU), and dynamic cache partitioning.

Expert Policies: SR-LRU and CR-LFU

Standard LRU and LFU have known failure modes under adversarial access patterns. CACHEUS introduces hardened versions that resist the two most common cache pollution patterns.

SR-LRU (Scan-Resistant LRU)

Standard LRU is vulnerable to scans: a sequential pass over many unique pages pollutes the entire cache. SR-LRU uses a two-stack architecture:

  • Q (probation): New pages enter here first. Pages hit in Q are promoted to S.
  • S (hot): Pages that have proven their value by being re-accessed. Only promoted pages live here.
  • Scan pages only touch Q once and never pollute S, providing natural scan resistance.

CR-LFU (Churn-Resistant LFU)

Standard LFU is vulnerable to churn: historically popular pages that are no longer relevant clog the cache with high frequency counts. CR-LFU addresses this via the shared heap:

  • The LFU heap spans both S and Q stacks
  • Eviction targets the global minimum frequency page
  • Demoted pages in Q retain their frequency but can be evicted by LFU if their frequency is globally low
  • New pages start with freq=1, naturally displacing stale high-frequency pages

Dynamic Cache Partitioning

def adjustSize(self, hit_in_Q):
    # Adapt partition sizes based on where hits occur
    if hit_in_Q:
        # Hits in Q mean S is too small -- expand S
        dem_count = max(1, self.dem_count)
        self.s_limit = min(
            self.cache_size - 1,
            self.s_limit + max(1, int((self.nor_count / dem_count) + 0.5)))
        self.q_limit = self.cache_size - self.s_limit
    else:
        # Hits in LRU history mean Q should be bigger
        nor_count = max(1, self.nor_count)
        self.q_limit = min(
            self.cache_size - 1,
            self.q_limit + max(1, int((self.dem_count / nor_count) + 0.5)))
        self.s_limit = self.cache_size - self.q_limit

Adaptive Learning Rate

CACHEUS's most impactful innovation is its gradient-based stochastic hill climbing for the learning rate. Instead of a fixed value, the learning rate continuously adapts based on the product of the learning rate change and hit rate change (the "delta").

Learning Rate Update Implementation

class Cacheus_Learning_Rate:
    def __init__(self, period_length):
        # Initial LR based on theoretical bound for 2-expert problem
        self.learning_rate = np.sqrt((2.0 * np.log(2)) / period_length)
        self.learning_rate_reset = min(max(self.learning_rate, 0.001), 1)
        self.hitrate_zero_count = 0
        self.hitrate_nega_count = 0

    def update(self, time):
        if time % self.period_len == 0:
            hitrate_curr = self.hitrate / self.period_len
            hitrate_diff = hitrate_curr - self.hitrate_prev
            delta_LR = self.learning_rate_curr - self.learning_rate_prev

            # Core: gradient sign detection
            delta = delta_LR * hitrate_diff
            delta = int(delta / abs(delta)) if delta != 0 else 0

            if delta > 0:       # Same direction worked: amplify
                self.learning_rate += abs(self.learning_rate * delta_LR)
                self.learning_rate = min(self.learning_rate, 1)
            elif delta < 0:    # Wrong direction: reverse
                self.learning_rate -= abs(self.learning_rate * delta_LR)
                self.learning_rate = max(self.learning_rate, 0.001)
            elif hitrate_diff <= 0:  # Stagnation: explore
                if self.hitrate_zero_count >= 10:
                    self.learning_rate = self.learning_rate_reset  # full reset
                else:
                    self.updateInRandomDirection()  # random perturbation
⚡
Initial Learning Rate Formula The initial learning rate is set to sqrt(2 * ln(2) / C) where C is the cache size. This comes from the theoretical optimal regret bound for the 2-expert multiplicative weights problem. For a cache of size 1000, this gives LR = 0.037, which is much more conservative than LeCaR's fixed 0.45.

Full Algorithm Architecture

Request Processing Flow

def request(self, oblock, ts):
    self.time += 1
    self.learning_rate.update(self.time)     # adapt LR every period

    if oblock in self.s:
        self.hitinS(oblock)                  # hit in hot stack: just update
    elif oblock in self.q:
        self.hitinQ(oblock)                  # hit in cold: PROMOTE to S
    elif oblock in self.lru_hist:
        evicted = self.hitinLRUHist(oblock)  # ghost hit: LRU was wrong
    elif oblock in self.lfu_hist:
        evicted = self.hitinLFUHist(oblock)  # ghost hit: LFU was wrong
    else:
        evicted = self.miss(oblock)          # cold miss: add to Q

Hit in Q: The Promotion Mechanism

def hitinQ(self, oblock):
    x = self.q[oblock]
    x.freq += 1                # increment frequency
    self.lfu[oblock] = x        # update in shared LFU heap

    if x.is_demoted:            # was previously demoted from S
        self.adjustSize(True)   # expand S since this was a useful page
        x.is_demoted = False
        self.dem_count -= 1

    del self.q[x.oblock]        # remove from Q
    self.q_size -= 1

    if self.s_size >= self.s_limit:
        y = self.s.popFirst()   # demote oldest from S to Q
        y.is_demoted = True
        self.dem_count += 1
        self.q[y.oblock] = y
        self.q_size += 1

    self.s[x.oblock] = x       # promote to S (MRU position)
    self.s_size += 1

Eviction: Weighted Expert Selection

def evict(self):
    lru = self.getLRU(self.q)     # SR-LRU candidate: oldest in Q
    lfu = self.getHeapMin()       # CR-LFU candidate: min freq in heap

    policy = self.getChoice()     # sample from W = [w_lru, w_lfu]

    if lru is lfu:
        evicted, policy = lru, -1  # same object: no history assignment
    elif policy == 0:
        evicted = lru
        del self.q[evicted.oblock]
        self.q_size -= 1
    elif policy == 1:
        evicted = lfu
        # LFU can evict from EITHER S or Q
        if evicted.oblock in self.s:
            del self.s[evicted.oblock]; self.s_size -= 1
        elif evicted.oblock in self.q:
            del self.q[evicted.oblock]; self.q_size -= 1

    del self.lfu[evicted.oblock]
    evicted.evicted_time = self.time
    self.addToHistory(evicted, policy)
    return evicted.oblock, policy

Weight Adjustment: Same as LeCaR Core

def adjustWeights(self, rewardLRU, rewardLFU):
    reward = np.array([rewardLRU, rewardLFU], dtype=np.float32)
    # Note: self.learning_rate is now a Cacheus_Learning_Rate object
    # __mul__ is overloaded to return learning_rate * other
    self.W = self.W * np.exp(self.learning_rate * reward)
    self.W = self.W / np.sum(self.W)
    # Clamp to [0.01, 0.99]
    if self.W[0] >= 0.99:
        self.W = np.array([0.99, 0.01], dtype=np.float32)

Internal Mechanisms and Design Details

Demotion and Promotion Tracking

CACHEUS tracks two critical counters for its dynamic partitioning: dem_count (pages demoted from S to Q) and nor_count (new pages in LRU history that were never promoted). The ratio of these counters drives the partition size adjustment.

# When a page is demoted from S to Q (S overflow)
y = self.s.popFirst()      # pop oldest from S
y.is_demoted = True        # mark as demoted
self.dem_count += 1         # increment demotion counter
self.q[y.oblock] = y       # add to Q
self.q_size += 1

# When a demoted page is hit in Q (was still useful!)
if x.is_demoted:
    self.adjustSize(True)   # S was too small -> expand S
    x.is_demoted = False
    self.dem_count -= 1

# When a new page enters LRU history without being promoted
if x.is_new:
    self.nor_count += 1     # this page never made it to S

Pollution Detection

CACHEUS includes a Pollutionator module that tracks cache pollution -- the fraction of unique pages in the cache that are accessed only once. High pollution indicates scan behavior or thrashing.

class Pollutionator:
    """Tracks the ratio of single-use pages in the cache.
    High pollution = scan workload, suggests SR-LRU should be aggressive.
    Updated every C requests."""
    def __init__(self, cache_size):
        self.cache_size = cache_size
        self.unique_count = 0       # pages seen only once
        self.unique_set = set()     # tracking set
        self.Y = []                 # pollution history

    def update(self, time):
        if time % self.cache_size == 0:
            pollution = self.unique_count / max(1, self.cache_size)
            self.Y.append(pollution)
            self.unique_count = 0

Design Philosophy CACHEUS demonstrates that the most effective ML-based systems are often not complex neural networks but rather carefully instrumented classical algorithms with learned parameters. The regret-based weight learning, adaptive learning rate, and scan/churn resistance are each simple ideas, but combined they produce a system that is robust across an enormous diversity of workloads -- without any manual tuning.

Hybrid Variants and Extensions

The CACHEUS repository includes experimental hybrid variants that combine the adaptive learning rate mechanism with other state-of-the-art cache algorithms, exploring whether the regret-learning framework can improve algorithms beyond LRU/LFU.

ARC + ALeCaR Hybrid

The arcalecar.py module combines ARC's four-list architecture (T1, T2, B1, B2) with ALeCaR's adaptive learning rate. Instead of ARC's rule-based parameter p adjustment, the hybrid uses regret-based weight learning to decide between the T1 (recency) and T2 (frequency) eviction targets.
