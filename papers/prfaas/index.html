2. Motivation — the Bandwidth Wall and the Hybrid-Attention Opening

2.1 Why RDMA-only PD disagg is an operational trap

RDMA-only PD constrains deployment location.

(1) $$\Phi_{\text{kv}}(l) \;=\; \dfrac{S_{\text{kv}}(l)}{T_{\text{prefill}}(l)}$$
(2) $$B_{\text{out}} \;\approx\; \dfrac{N}{P}\cdot \Phi_{\text{kv}}(L_{\text{avg}})$$

~3.8 Tbps egress requirement locks prefill on-premise.

2.2 Hybrid attention shifts the deployable boundary

Hybrid attention: linear (fixed state) + sparse full-attention (block KV).

Why is this not trivial? Smaller KV makes mean work; burstiness/skew make second-order effects matter.

4. Hybrid Prefix Cache Pool

Hybrid attention creates two kinds of reusable state per request.

Linear-attention layers

  • Request-level recurrent state
  • Fixed size
  • Per-conversation reuse only

Full-attention layers

  • Block-level KV
  • Length-dependent
  • Cross-request reusable

4.1 Unified block pool with two categories

Single allocator backing both; each block tagged with category.

Figure 4 (from paper): Hybrid prefix cache pool: prefix-cache vs. transfer-cache blocks.

Prefix-cache block

  • Fully populated, cross-request reusable
  • Never crosses WAN
  • Drives cache-aware placement
▼

Transfer-cache block

  • Tail KV needing handoff
  • One-shot, discarded after handoff
  • Size ∝ $l$, not full context
Why unified allocator matters: Unified allocator avoids cross-pool starvation under bursty workloads.

5. Bandwidth-Aware Scheduling

5.1 Throughput model

Three queues: PrfaaS-long-prefill, PD-P-short-prefill, PD-D-decode.

(3) PrfaaS $$\Theta_{\text{prfaas}} \;=\; \min\!\left(\dfrac{N_{\text{prfaas}}}{T_{\text{prefill}}(l_{\text{long}})},\; \dfrac{B_{\text{out}}}{S_{\text{kv}}(l_{\text{long}})}\right)$$
(4) PD-P $$\Theta_{\text{pd-p}} \;=\; \dfrac{N_p}{T_{\text{prefill}}(l_{\text{short}})}$$
(5) PD-D $$\Theta_{\text{pd-d}} \;=\; \dfrac{N_d \cdot BS_{\max}}{T_{\text{decode}} \cdot L_{\text{out}}}$$
(6) System $$\Lambda_{\max} \;=\; \min\!\left(\dfrac{\Theta_{\text{prfaas}}}{p},\; \dfrac{\Theta_{\text{pd-p}}}{1-p},\; \Theta_{\text{pd-d}}\right)$$

System throughput = bottleneck of three queues.

5.2 Optimality conditions

Maximizing $\Lambda_{max}$ requires two balance equations:

(7) Producer balance $$\dfrac{\Theta_{\text{prfaas}}}{p} \;=\; \dfrac{\Theta_{\text{pd-p}}}{1-p}$$
(8) Producer/consumer match $$\Theta_{\text{prfaas}} + \Theta_{\text{pd-p}} \;=\; \Theta_{\text{pd-d}}$$

Two knobs: threshold $t$ and split $N_p / N_d$.

Figure 5a: Grid search over $N_p$ vs. $N_d$.

5.3 Two-timescale online scheduler

Fast loop — bandwidth & cache aware routing

  • Tightens $t$ near link ceiling
  • Picks cluster with longest local prefix hit under bandwidth pressure
  • Can migrate bulk prefix when bandwidth abundant

Slow loop — traffic-driven reallocation

  • Identifies binding constraint via queue depth
  • Role-switches GPUs inside PD cluster
  • Re-solves for $t$
Why two timescales: Bandwidth fluctuates per-request; workload shape drifts per-hour — two loops match two frequencies.

7. Discussion, Limitations, and Where PrfaaS Fits

7.1 What this paper proves vs. what it does not

Proven

  • ✓ WAN PD disagg numerically feasible for hybrid models
  • ✓ Selective routing beats both baselines on one workload
  • ✓ ~87% link headroom at measured load

Not proven

  • ◐ Dense-attention models — out of scope by premise
  • ◐ Real WAN loss/jitter — limited stress
  • ◐ Multi-tenant fairness
  • ◐ Cross-DC failure modes

7.3 Takeaways for agentic serving

Agentic workloads amplify PrfaaS's axes, but retention policy during tool-call pauses is still unaddressed.
