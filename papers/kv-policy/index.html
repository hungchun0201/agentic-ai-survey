Learning to Evict from Key-Value Cache

arXiv:2602.10238

Per-head RL agents that learn to rank KV cache entries by predicted future utility — replacing heuristic eviction with a principled, query-independent, budget-agnostic policy.

Overview

As LLM context windows grow into the hundreds of thousands of tokens, the KV cache — the mechanism that stores previously computed key and value vectors to avoid redundant computation — becomes a critical memory bottleneck. Existing eviction strategies rely on heuristic proxies such as recency, attention sinks, or past attention scores. KV Policy (KVP) reframes eviction as a reinforcement learning ranking problem: a lightweight per-head agent learns to score every cache entry by its predicted future utility, enabling eviction at any budget by simply discarding the lowest-ranked entries.

Problem Formulation

Given n tokens in the cache and a budget b, select the subset of size b that maximizes downstream performance. Under two mild conditions — uniqueness (each optimal subset is unique) and nestedness (S*_b is a subset of S*_{b+1}) — the authors prove this is equivalent to learning a single total order (ranking) over all tokens:

S*_b = argmax_{|S|=b, S ⊂ X} R(S)     ⇔     S*_b = { i ∈ [n] : π(i) ≤ b }   for all b
Key Insight — Proposition 1

Under uniqueness and nestedness, KV cache eviction reduces to KV cache ranking. A single budget-agnostic scoring function whose top-b elements realize S*_b for every b is guaranteed to exist. This converts an NP-hard subset selection into a tractable learning-to-rank problem.

Why Heuristics Fail

Methods like StreamingLLM, TOVA, SnapKV, and H2O rely on backward-looking proxies: past attention scores, recency, or attention sinks. These assume what was important before will remain important, an assumption that breaks when informational needs shift across generation steps. A token critical for future steps may have received zero past attention. KVP learns a forward-looking, query-independent policy that directly predicts future utility from only keys, values, and position.

Agent Architecture

KVP Per-Head Agent Architecture

Scoring Function

For each token x_i, the representation is the concatenation of its key vector k_i, value vector v_i, and original position pos_i. The scoring function is a small MLP:

// Per-head scoring function (no attention info needed)
x_i = concat(k_i, v_i, pos_i)
score = MLP_θ(x_i)       // 2-layer, 256 hidden units, ~650K params

// Plackett-Luce ranking via Gumbel-Sort (fully parallel)
g_i ~ Gumbel(0, 1)       // i.i.d. noise
σ  = argsort(score + g_i) // one forward pass + sort
Design Advantage

The agent uses only keys, values, and positions — no queries, no attention scores, no future information. This makes it query-independent and compatible with FlashAttention, unlike TOVA or SnapKV which require recomputing attention statistics at inference time.

Efficient Offline Training

Training Pipeline

Training Details

  • Architecture: Qwen2.5-7B-Chat with Grouped-Query Attention (GQA) — 4 KV heads across 28 layers = 112 agents
  • Agent size: 2-layer MLP with 256 hidden units (~650K parameters each, 2.6MB checkpoint)
  • Optimizer: AdamW, learning rate 5×10-5, cosine schedule with 100-step warmup
  • Variance reduction: REINFORCE with Leave-One-Out (RLOO) baseline
  • Gradient clipping: max norm 5, advantages normalized by mean and std
  • Training time: Under 30 minutes on a single node of 8 NVIDIA H100 GPUs
No LLM in the Loop

A critical advantage: the reward signal is computed from pre-stored attention patterns, so training requires zero additional LLM forward passes. This makes KVP training vastly cheaper than methods requiring online LLM evaluation.

Holistic Reward Signal

The reward globally evaluates ranking quality across all possible cache budgets simultaneously. For a candidate ranking σ (most to least important), the total reward sums per-budget rewards:

R(σ_1, ..., σ_n; X) = Σ_{b=1}^{n-1} R^b(σ_{n-b}, ..., σ_n; X)

Each per-budget reward R^b measures the negative total future attention accumulated by evicted tokens:

R^b = - Σ_{i=b+1}^{n} Σ_{j=n+1}^{n+f} A(x_{σ_i}, x_j)

where A(x_i, x_j) is the future attention from token x_j to cached token x_i. The reward is normalized by the cost of an optimal ranking σ* to create a scale-invariant objective.

Why This Reward Works

Budget-Agnostic

By summing over all budgets 1..n-1, a single ranking is trained to be good at every possible compression ratio. No need to re-train for different memory constraints.

No Extra Inference

Future attention patterns are precomputed during data generation. The reward is evaluated purely from stored tensors, avoiding the cost of running the LLM during RL training.

Direct Optimization

Unlike supervised surrogates that must approximate rankings through differentiable relaxations, the RL objective directly optimizes the non-differentiable sort operation via policy gradient.

Ablations

Is RL Necessary? (vs. Supervised Surrogates)

The authors compare RL training against a supervised baseline using a differentiable sorter with MSE loss on ground-truth ranks. The supervised agent fails to learn an effective policy, exhibiting high and unstable cost curves. The RL agent reaches low, stable cost.

Why RL Wins

KV eviction is inherently sparse: only a small fraction of tokens are critical. Differentiable sorting surrogates face a temperature trade-off — low temperature gives vanishing gradients, high temperature incorrectly disperses importance. Policy gradient methods bypass this by providing an unbiased learning signal through the non-differentiable sort, concentrating credit on the few ranking decisions that matter.

Per-Head Policy Diversity

Analysis reveals that different heads learn strikingly different eviction patterns. A fixed heuristic like StreamingLLM may work well for certain heads (e.g., layer 22, head 0) but be catastrophic for others (layer 19, head 0). Even the strong attention-free heuristic KeyDiff exhibits hard failure modes on specific heads. This validates the per-head specialization design.

Reward Effectiveness

Measuring eviction cost (-R^b) per budget on unseen test data shows KVP performs on par with privileged attention-aware methods (TOVA, SnapKV) despite not using query information — confirming that offline RL training successfully distills attention-based ranking principles into an efficient static policy.

Relevance to Agentic AI Systems

1

Long-Context Agent Memory

Agents processing extended conversations, code files, or multi-step reasoning chains accumulate massive KV caches. KVP enables efficient compression without sacrificing retrieval of critical information scattered across the context.

2

Per-Head = Per-Capability

Different attention heads specialize in different information types (syntax, semantics, entity tracking). KVP's per-head agents naturally adapt to these diverse agentic information needs, preserving what each capability requires.

3

Complementary to Offloading

KVP's learned rankings provide a principled signal for memory hierarchy management: lowest-ranked entries are natural candidates for offloading to CPU RAM or disk, enabling tiered caching strategies for resource-constrained agents.

4

Dynamic Budget Allocation

Future work on non-uniform budgets across heads and layers could enable agents to dynamically allocate memory to the capabilities most needed for the current task — more memory for reasoning heads when solving math, more for retrieval heads when searching.

Agentic Paradigm Shift

KVP demonstrates that learned, adaptive policies outperform hand-crafted heuristics for cache management — a principle that extends naturally to agentic systems where information needs are diverse, dynamic, and impossible to capture with fixed rules. The per-head RL framework offers a template for building specialized micro-agents that collectively optimize complex system-level objectives.

Algorithm Summary

// Algorithm 1: RL Training on Pre-Computed KV Traces
Input: Static dataset of m sequences, each with Q, K, V tensors
while not converged do
    j ~ Uniform(1, m)           // sample sequence
    n ~ Uniform(1, n_j)          // sample cache size

    for k = 1 to K do            // K parallel episodes
        σ^k ~ π_θ(x_1, ..., x_n) // sample permutation via Gumbel-Sort
    end

    Compute R(σ^k)              // reward: sum of per-budget future attention costs
    R_bar = mean(R(σ^k))        // RLOO baseline

    // Policy gradient update
    ∇J(θ) ≈ 1/K Σ_k (R(σ^k) - R_bar) Σ_i ∇ log π_θ(σ^k_i | X)
    θ ← AdamW(θ, ∇J)
end while

// Inference: rank entries, evict bottom (n - budget) tokens
scores = MLP_θ(k_i, v_i, pos_i)  // per entry
keep = top_b(scores)              // retain top-b by score

Future Directions

  • Adaptive Budget Allocation: Dynamically assign different cache budgets to different heads and layers based on task requirements.
  • Inter-Head Coordination: Joint optimization across heads to capture cross-head dependencies.
  • Online Adaptation: Fine-tune agents on-the-fly during deployment for domain-specific optimization.
  • Integration with Quantization: Combine ranking-based eviction with low-rank or quantized representations for multiplicative memory savings.
  • Multi-Agent Collaboration: Extend the per-head agent paradigm to coordinate across layers for hierarchical memory management.

Why Differentiable Sorting Fails

The authors ablated against a supervised baseline using the differentiable sorter from Blondel et al. (ICML 2020). This approach replaces the policy gradient objective with a length-normalized MSE loss between predicted soft ranks and ground-truth ranks. The supervised agent fails because KV cache eviction is inherently sparse: only a small fraction of tokens carry critical future attention. Differentiable surrogates face a fundamental temperature dilemma:

  • Low temperature: Sharp soft ranks give vanishing gradients for most tokens, preventing learning
  • High temperature: Smooth soft ranks give dense but biased gradients that incorrectly disperse importance across irrelevant tokens

Policy gradient methods bypass this by operating on the non-differentiable sort directly. The REINFORCE algorithm with RLOO baseline concentrates the learning signal on the few ranking decisions that actually matter, enabling direct credit assignment.

Complementary Techniques

Quantization (MiniKV, SmoothQuant)

Reduces the memory footprint per KV entry (e.g., 8-bit or 2-bit). Orthogonal to eviction: KVP first identifies which tokens to keep, then quantization compresses the retained entries further.

Low-Rank Compression (Palu, LOKI)

Represents K/V matrices using lower-dimensional projections. KVP can pre-filter tokens before projection, reducing the matrix dimensions and improving compression quality.

State Merging (MorphKV)

Identifies and combines similar KV pairs into compact representations. KVP's ranking can first filter the least useful tokens, then MorphKV merges the remaining high-utility entries for additional savings.

Memory Hierarchies (IceCache, ArkVale)

Offloads entries to CPU RAM or disk. KVP provides a principled signal for tier placement: highest-ranked entries stay on GPU, mid-ranked go to CPU, lowest are evicted entirely.

Inference Protocol

At inference, the learned policy ranks all tokens except the first 4 and last 16 (which are always retained). Token eviction is emulated via custom attention masks in FlexAttention, preventing the model from attending to pruned tokens during generation. This approach requires no modification to the underlying LLM weights or architecture — only the attention mask is changed.

// Inference-time eviction (per KV head)
scores = kvp_agent(keys, values, positions)  // MLP forward pass
keep_idx = topk(scores, budget)                   // select top-b entries
keep_idx = union(keep_idx, first_4, last_16)      // always retain boundaries
attn_mask = build_flex_mask(keep_idx)              // FlexAttention mask
output = generate(model, prompt, attn_mask)       // standard generation

Deployment Considerations

Integrating KVP into a production inference stack requires attention to several practical dimensions beyond raw accuracy. The following covers memory footprint, compatibility, and serving strategies.

Because KVP agents are completely decoupled from the base LLM weights, they can be updated or swapped independently — for example, a domain-specific agent set can replace the general-purpose one without touching the LLM itself. This makes KVP compatible with continuous delivery workflows where model and compression policy are versioned separately.

Framework Compatibility

F

FlexAttention (PyTorch)

KVP uses FlexAttention custom masks to emulate eviction without modifying the KV cache tensor itself. This avoids memory reallocation at inference time and enables zero-copy eviction.

V

vLLM / SGLang Integration

KVP's query-independent design is compatible with prefix caching and paged attention. The MLP scoring pass fits naturally into the prefill phase before block allocation, making integration with PagedAttention schedulers straightforward.

Q

Quantization Stack

KVP agents themselves can be quantized to INT8 with negligible accuracy loss, reducing the 291 MB agent overhead to under 75 MB. This is especially beneficial on edge devices with tight memory budgets.

B

Batched Inference

Because all 112 MLP agents score tokens independently and in parallel, batched scoring adds negligible latency even at large batch sizes. The scoring overhead scales sub-linearly with batch size due to GPU parallelism.

Steps to Adapt KVP to a New LLM

  1. Run the target LLM on a representative sample of sequences (e.g., 5K–10K examples from your deployment domain) and store Q, K, V tensors at every layer and head.
  2. Instantiate one 2-layer MLP per KV head, sized to match the model's head dimension (d_k, d_v). For GQA models, one agent serves multiple query heads sharing the same KV head.
  3. Run RL training with REINFORCE+RLOO using the holistic reward (sum over all budgets of negative future attention cost). Training takes under 30 minutes on 8xH100 for a 7B-scale model.
  4. Validate on a held-out set covering your target tasks. Check cost curves per head to confirm all agents converge; heads with persistent high cost may need more training data or a wider MLP.
  5. Integrate the scored rankings into your serving engine by replacing the standard attention mask with a FlexAttention mask that zeros out evicted token positions.
  6. Profile end-to-end latency at representative context lengths (e.g., 2K, 4K, 8K tokens) and cache budgets (25%, 50%, 75%) to confirm the <1% prefill overhead claim holds for your hardware configuration.
Eviction Boundary Heuristic

KVP always retains the first 4 tokens (system prompt tokens that anchor the model's behavior) and the last 16 tokens (the most recent context, which is universally important for coherent generation). Only the middle tokens are ranked and selectively evicted. This boundary heuristic is consistent with prior work showing attention sinks at position 0 and a recency bias in autoregressive generation.

Key Takeaways

KV Policy makes several contributions that collectively shift how we think about KV cache management in long-context LLMs and agentic systems. The six points below capture the most transferable insights for practitioners.

1

Eviction as Ranking

The theoretical result (Proposition 1) that uniqueness and nestedness imply a total order is a clean reduction: the NP-hard subset selection collapses to a standard ranking problem solvable with existing learning-to-rank machinery.

2

RL over Supervised Learning

The sparsity of critical tokens makes differentiable surrogates fundamentally ill-suited. Policy gradient methods handle non-differentiable objectives natively, concentrating learning on the few tokens that decide outcome quality.

3

Per-Head Specialization Matters

Global heuristics fail because different heads encode entirely different types of information. KVP's ablations show that a single policy across all heads degrades significantly; 112 specialized agents are necessary for strong performance.

4

Offline Training is Sufficient

Precomputing KV traces once and training on stored tensors achieves parity with online methods, while cutting compute cost by orders of magnitude. This makes KVP practical to retrain whenever a new base model is deployed.

5

Zero-Shot Domain Transfer

KVP agents trained on RULER or OASST2 generalize to diverse out-of-distribution benchmarks (BoolQ, MMLU, HellaSwag, GovReport) with minimal accuracy loss, suggesting the learned ranking principles capture task-agnostic attention structure.

6

Composability with Other Techniques

Because KVP outputs a clean priority ranking, it composes naturally with quantization, low-rank compression, and memory tiering. The ranking signal tells downstream techniques which entries deserve the highest fidelity, enabling compound compression pipelines.

Limitations to Keep in Mind

KVP requires retraining agents whenever the base LLM changes, since agent weights are tied to the specific key/value dimensionality and learned representations of a given model. Additionally, the current evaluation focuses on English-language benchmarks; performance on multilingual or code-heavy workloads remains an open question. Finally, the assumption of nestedness (S*_b ⊂ S*_{b+1}) holds empirically but is not theoretically guaranteed for all tasks.

Despite these caveats, KVP represents a meaningful step toward principled, adaptive KV cache management — and the offline RL training approach is practical enough to deploy alongside today's LLM serving stacks.
