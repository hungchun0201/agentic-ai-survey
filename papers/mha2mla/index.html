Overview

The KV Cache Memory Crisis

Standard Multi-Head Attention (MHA) stores full-rank K and V tensors for every token across all heads, creating a KV cache that scales as O(2 * l * n_h * d_h), where l is sequence length, n_h is the number of heads, and d_h is the head dimension. For a 7B Llama2 model with 32 heads and d_h=128, this means 256 dimensions per token per layer. At a 128K context length, the KV cache can consume over 60GB of GPU memory -- often exceeding the model weights themselves.

DeepSeek's Multi-Head Latent Attention (MLA) compresses the KV cache into a low-rank latent vector c_kv of dimension d_kv, achieving O(l * (d_r + d_kv)) where (d_r + d_kv) is much smaller than 2 * n_h * d_h. However, MLA requires training from scratch, making it inaccessible for existing MHA/GQA models.

MHA2MLA: The First Retrofit Framework

MHA2MLA is the first data-efficient fine-tuning framework that adapts pre-trained MHA or GQA models to the MLA architecture. The key research question: Can LLMs originally well-trained for MHA be adapted to enable MLA for inference?

The answer is yes, through two carefully designed techniques:

  • Partial-RoPE removal: Strategically convert selected RoPE dimensions to NoPE (No Positional Encoding), creating the position-agnostic subspace that MLA requires for latent compression
  • Joint SVD approximation: Apply SVD on concatenated pre-trained W_k and W_v to initialize the low-rank down-projection and up-projection matrices, preserving learned knowledge

Key Contributions

  • First parameter-efficient framework to adapt pre-trained MHA/GQA-based LLMs to MLA using only 0.6-1% of training data
  • Demonstrates compatibility with KV cache quantization for compound gains up to 96.87% KV reduction
  • Experiments across five model sizes (135M to 13B), covering both MHA and GQA architectures
  • Detailed ablation studies comparing four partial-RoPE strategies and two SVD factorization methods

Background: MHA, GQA, and MLA

Multi-Head Attention (MHA)

For each head h, MHA projects input x_i into queries, keys, and values:

q_i = x_i * W_q,   k_i = x_i * W_k,   v_i = x_i * W_v

After applying RoPE to q and k, scaled dot-product attention is computed. The KV cache stores all {k_rope, v} pairs per head, totaling O(2 * l * n_h * d_h) per layer.

Variants: GQA shares keys/values across n_g groups (n_g is much less than n_h), reducing KV cache to O(2 * l * n_g * d_h). MQA is GQA with n_g = 1. Both sacrifice parameters for memory savings.

Multi-Head Latent Attention (MLA)

MLA decouples positional encoding from KV compression. The input x_i is split into two streams:

Position-Aware (RoPE): A small d_r-dimensional subspace retains RoPE for positional sensitivity. These k_rope vectors must be cached alongside the latent.

Position-Agnostic (NoPE): The remaining dimensions are compressed into a shared latent vector c_kv = x_i * W_dkv. During inference, k_nope and v are reconstructed via c_kv * W_uk and c_kv * W_uv.

KV cache per token = d_r + d_kv   (vs. 2 * n_h * d_h for MHA)
Why does MLA need separate RoPE and NoPE? For the NoPE portion, the identity q_nope * k_nope^T = x_i * (W_dq * W_qc * W_uk^T) * c_kv^T allows pre-merging the query-side matrices into a single matrix. But for the RoPE portion, the rotation matrix R_{j-i} depends on relative position, preventing such merging. MLA therefore preserves RoPE in only a small subspace for position awareness while enabling aggressive compression in the rest.

Key Mechanism 1: Partial-RoPE Strategies

MHA applies RoPE to all d_h dimensions of queries and keys. MLA requires only a small d_r-dimensional subspace to retain RoPE while the rest becomes NoPE for latent compression. MHA2MLA must bridge this gap by selectively removing RoPE from most dimensions.

Given r retained rotational subspaces (each 2D, so d_r = 2r) out of d_h/2 total subspaces, the paper evaluates four strategies for choosing which subspaces keep their positional encoding:

S_high: High-Frequency Preservation

Retains the r fastest-rotating (highest frequency) subspaces: S_high = {k | 0 ≤ k < r}. High-frequency components rotate rapidly at ~1 radian per token. This is consistent with p-RoPE explored by Barbero et al. (2024).

Result: -0.82% on 135M, -0.91% on 1B7. Performs well because high-frequency subspaces carry the most positional signal.

S_low: Low-Frequency Preservation

Retains the r slowest-rotating (lowest frequency) subspaces. Low-frequency components rotate at ~beta^(1/d_h) radians per token.

Result: -5.25% on 135M, -1.87% on 1B7. Worst performer. Removing high-frequency components destroys the attention's ability to distinguish nearby positions, critical for autoregressive generation.

S_uniform: Uniform Sampling

Selects r subspaces at equidistant intervals across the frequency spectrum: S_uniform = {floor(k * d_h / 2r) | 0 ≤ k < r}. Balances high and low frequencies through geometric spacing.

Result: -0.41% on 135M, -0.41% on 1B7. Strong overall but does not account for head-specific frequency importance.

S_2-norm: Contribution-Aware Selection (Default)

For each head h, compute the mean 2-norm of q_k and k_k across a calibration dataset. Rank subspaces by their contribution to attention logits (upper-bounded by Cauchy-Schwarz). Select top-r per head.

S_2-norm = top-r ( ||q*[2k,2k+1]|| * ||k*[2k,2k+1]|| ) for 0 ≤ k < d_h/2

Result: -0.69% on 135M, -0.73% on 1B7. Slightly worse than S_uniform alone, but significantly better when combined with SVD, because the removed NoPE dimensions are more suitable for SVD compression.

Key Mechanism 2: Low-Rank SVD Approximation

After partial-RoPE removes positional encoding from most dimensions, the NoPE portions of W_k and W_v can be compressed into a shared low-rank latent space. MHA2MLA proposes two SVD-based strategies:

SVD_split: Decoupled Factorization

Separately decompose W_k,nope and W_v using truncated SVDs, each allocating d_kv/2 dimensions:

W_k,nope = U_k * Sigma_k * V_k^T    W_v = U_v * Sigma_v * V_v^T

The down-projections and up-projections become:

  • W_dk = U_k * Sigma_k^(1/2),   W_uk = Sigma_k^(1/2) * V_k^T
  • W_dv = U_v * Sigma_v^(1/2),   W_uv = Sigma_v^(1/2) * V_v^T

Each latent is constructed as c_kv = [x * W_dk, x * W_dv], requiring two separate down-projections.

SVD_joint: Joint Factorization (Default)

Jointly factorize the concatenated matrix [W_k,nope; W_v]:

[W_k,nope ; W_v] = U_kv * Sigma_kv * V_kv^T

This yields a single shared down-projection:

  • W_dkv = U_kv * Sigma_kv^(1/2)
  • W_uk = Sigma_kv^(1/2) * V_kv[:, :d_v]
  • W_uv = Sigma_kv^(1/2) * V_kv[:, d_v:]

The single latent c_kv = x * W_dkv preserves cross-parameter dependencies between keys and values, leading to better reconstruction quality.

# Joint SVD implementation from the repo (patch_func.py) def svd_low_rank_approx(k_c_weight, v_weight, d_kv_mid, method="joint"): if method == "joint": joint_kv = torch.cat([k_c_weight, v_weight]) # [d_k+d_v, d_in] down_kv, up_kv = SVD(joint_kv, d_kv_mid) # truncated SVD up_k, up_v = up_kv.split([d_k_c, d_v]) # split up-projection kv_proj = LowRankKVLinear( d_in, d_k_c, d_v, d_mid=d_kv_mid, k_approx=True, v_approx=True, kv_joint=True ) kv_proj.reset_parameters(down_kv_weight=down_kv, up_k_weight=up_k, up_v_weight=up_v) return kv_proj # LowRankKVLinear forward: inference-time computation def mha_forward(self, x): c_kv = self.down_kv(x) # x @ W_dkv -> shared latent [bsz, seq, d_kv] k = self.up_k(c_kv) # c_kv @ W_uk -> NoPE keys v = self.up_v(c_kv) # c_kv @ W_uv -> values return k, v
SVD_joint vs SVD_split: Joint factorization outperforms split by 0.91% (135M) and 1.38% (1B7) on average across benchmarks. The advantage comes from jointly optimizing the latent space for both keys and values, retaining cross-parameter correlations that split SVD destroys.

Why S_2-norm Wins with SVD

While S_uniform slightly outperforms S_2-norm for partial-RoPE alone, the gap reverses when SVD is applied. S_2-norm selects dimensions by their actual contribution to attention logits, which means the removed NoPE dimensions are those with the least impact -- precisely the dimensions most suitable for aggressive SVD compression. This synergy between contribution-aware selection and low-rank approximation is a key design insight.

Hyperparameter r (RoPE Dimensions)

The number of retained RoPE subspaces r creates a tradeoff: higher r preserves more positional information but reduces the compressible NoPE subspace. Default r = d_h/16 (e.g., r=4 for d_h=128) was found optimal across model sizes. At r=d_h/8, compression efficiency drops without significant accuracy gain. At r=d_h/32, position sensitivity is impaired for longer sequences.

Key Implementation Detail: Custom RoPE

The patching_llama.py file monkey-patches the HuggingFace LlamaAttention forward pass. A custom apply_rotary_pos_emb function is created per-layer using closure-captured index tensors (q_r_indices, k_r_indices). This approach accesses the layer_idx via stack frame inspection to apply the correct per-head, per-layer RoPE mask.

LowRankKVLinear Module

The core nn.Module stores either separate down_k/down_v projections (split mode) or a shared down_kv (joint mode), plus up_k and up_v for reconstruction. The mha_forward method computes k,v through the bottleneck. A mla_forward method exists as a stub for future Triton-optimized inference that would cache only c_kv.

# Patched Llama attention forward (simplified from patching_llama.py) def custom_LlamaAttention_forward(self, hidden_states, position_embeddings, ...): # NoPE path: compute via low-rank KV linear key_c_states, value_states = self.kv_proj.mha_forward(hidden_states) # RoPE path: separate small projection for positional keys key_r_states = self.k_r_proj(hidden_states) # d_r dimensions only query_states = self.q_proj(hidden_states) # Split query into RoPE and NoPE parts query_r_states = query_states[..., :n_heads * d_r] query_c_states = query_states[..., n_heads * d_r:] # Apply RoPE only to the r-dimensional subspace cos, sin = position_embeddings query_r_states, key_r_states = apply_rotary_pos_emb(query_r_states, key_r_states, cos, sin) # Concatenate RoPE and NoPE for standard attention query_states = torch.cat([query_r_states, query_c_states], dim=-1) key_states = torch.cat([key_r_states, key_c_states], dim=-1) # ... standard attention computation follows ...

Significance and Limitations

Democratizing MLA

MLA was exclusive to DeepSeek models trained from scratch. MHA2MLA enables any MHA or GQA model (Llama, Qwen, Phi, SmolLM) to adopt MLA's KV cache efficiency through lightweight fine-tuning, opening the architecture to the broader ecosystem of tens of thousands of pre-trained models.

Compound Efficiency

MHA2MLA is orthogonal to quantization and pruning. The combination MHA2MLA + Int4 achieves 96.87% KV reduction without destructive interference -- a level of compression that no single technique achieves alone. This multiplicative composability is a significant practical advantage.

Favorable Scaling

The accuracy degradation decreases with model scale: 2.24% at 135M, 0.30% at 7B, 0.23% at 13B. This suggests that MHA2MLA will be even more effective on larger models (70B+), where KV cache savings matter most.

Practical Accessibility

Fine-tuning requires only 0.6-1% of pre-training data and 4-36 hours on NVIDIA L20Y GPUs. The open-source implementation supports SmolLM (135M-1B7), Llama2 (7B-13B), Qwen1.5, Qwen2, and Qwen3 families with detailed configuration files and pre-computed 2-norm ranks.

Limitations: (1) Not yet validated on models larger than 13B due to compute constraints; Llama3-scale models require 128K context fine-tuning. (2) DeepSeek has not open-sourced tensor-parallel MLA inference kernels, making deployment at scale challenging. (3) The paper focuses on data efficiency; parameter-efficient approaches (freezing FFN, freezing RoPE-retained parameters) are left for future work. (4) The current implementation expands the latent during attention rather than caching only c_kv, meaning full KV cache savings require additional inference engine support.

Related Work and Context

Efficient Attention Architectures

MHA remains foundational but its quadratic complexity in context length has spurred many alternatives. Multi-Query Attention (MQA, Shazeer 2019) and Grouped-Query Attention (GQA, Ainslie et al. 2023) reduce memory by sharing keys/values across heads, at the cost of parameter pruning and some quality degradation. Linear Transformers (Katharopoulos et al. 2020), RWKV (Peng et al. 2023), and Mamba (Gu and Dao 2023) replace softmax attention entirely but struggle to match MHA expressiveness in autoregressive generation. MLA (DeepSeek-AI 2024) uniquely compresses KV caches into low-rank latent vectors without pruning attention parameters, preserving quality while reducing memory. MHA2MLA bridges MLA with mainstream architectures.

KV Cache Compression Techniques

Quantization: GPTQ (Frantar et al. 2022), FlexGen (Sheng et al. 2023), and KIVI (Liu et al. 2024) store KV caches in low-bit formats (2-4 bit), trading precision for memory. MHA2MLA is orthogonal to these and compounds with them.

Dynamic Pruning: A2SF (Jo and Shin 2024) and SnapKV (Li et al. 2024) prune less-important tokens from the cache. However, token pruning risks discarding critical long-range dependencies.

Head Pruning: SliceGPT (Ashkboos et al. 2024), Sheared (Xia et al. 2024), and Simple Pruning (Sun et al. 2024) irreversibly reduce model capacity. MHA2MLA preserves all heads while compressing their representation.

Inference Optimization: MLA's Matrix Merging

A key advantage of MLA that MHA2MLA inherits is the ability to pre-merge query-side matrices for NoPE attention. During inference, the NoPE attention computation becomes:

# Standard NoPE attention (naive) q_nope = x @ W_q_nope # [bsz, seq, d_nope] k_nope = c_kv @ W_uk # [bsz, cache_len, d_nope] attn = q_nope @ k_nope.T # O(seq * cache_len * d_nope) # Optimized: pre-merge W_q_nope @ W_uk^T into single matrix W_merged = W_q_nope @ W_uk.T # precomputed once c_q = x @ W_merged # [bsz, seq, d_kv] attn = c_q @ c_kv.T # O(seq * cache_len * d_kv) # d_kv << d_nope, so this is much faster

This matrix merging reduces the attention FLOPs proportionally to d_kv/d_nope, providing not just memory but also compute savings. The RoPE portion cannot be merged because the rotation matrix depends on relative position (j-i), so it must be computed in the original form.

Future Directions

Parameter-Efficient Fine-Tuning

The current approach uses full-parameter fine-tuning. Future work could explore freezing the FFN module (which is untouched by the MHA-to-MLA transformation) and/or freezing the RoPE-retained dimensions of Q and K projections. This could reduce GPU memory requirements during training and potentially allow fine-tuning on consumer hardware.

Larger Model Validation

The paper covers models up to 13B. Extending to Llama3 (8B-70B), Qwen2.5 (72B), and Mixtral (8x7B MoE) would validate the scaling law and demonstrate practical value at production scale. The recently added Qwen3 support in the repo suggests active development toward broader coverage.

Native MLA Inference Engines

To realize the full KV cache savings, inference engines must cache only c_kv + k_rope instead of the expanded K and V. This requires changes to frameworks like vLLM, TensorRT-LLM, and SGLang to support MLA-style KV cache management. The repo includes a Triton kernel stub (mla_triton_kernel.py) pointing toward this direction.

Integration with Other Compression

The paper demonstrates composability with Int4 quantization. Further combinations with token-level pruning (SnapKV), layer-level sharing (MiniCache), and speculative decoding could yield even greater efficiency gains. The orthogonality of MHA2MLA's latent space to these techniques is a strong foundation for compound optimization.
