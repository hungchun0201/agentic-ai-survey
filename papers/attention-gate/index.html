In-context KV-Cache Eviction
for LLMs via Attention-Gate

Overview

Attention-Gate (AG) introduces a lightweight, trainable module placed before each multi-head attention (MHA) layer that generates per-token binary eviction flags. Unlike prior heuristic methods (StreamingLLM, H2O) that rely on accumulative attention scores or fixed windows, AG uses global context to decide which tokens to keep or discard -- adapting independently across layers, heads, tokens, and tasks.

Core Method: Attention-Gate

Limitations of Existing Approaches

Lack of Flexibility

StreamingLLM uses a fixed window + initial tokens. It cannot adapt to different contexts -- e.g., "cute" matters for sentiment, "cat" matters for object recognition.

Attention Bias

H2O uses accumulative attention scores but suffers from attention bias -- over-prioritizing initial or recent tokens, especially in long-context inputs.

No Global Context

Methods like NACL, A2SF rely on local statistics of attention. They cannot capture token importance that depends on the full context.

Attention-Gate Architecture

AG consists of two components: (i) a reduced multi-head attention module (MHA') for global information exchange, and (ii) a Sigmoid-based gating mechanism (G) for threshold-based eviction.

Mathematical Formulation

AG(X) = G( MHA'(X), τ ) ∈ {0, 1}n × h
G(s, τ) = 1 if Sigmoid(s) > τ, else 0

The mask for attention scores of head Hi integrates AG flags with the causal mask:

mi(j,t) = { 0 if j < t; 1 if j = t; AG(X)i(t) otherwise }
Diagonal Preservation: Even when a token is evicted from other positions, its self-attention (diagonal) is always preserved. This ensures each token always interacts with its own key, maintaining self-attention integrity.

Architecture Details

Design Choices

Why Sigmoid over Softmax? Sigmoid computes probabilities independently for each token, which is critical for eviction -- a token's importance should not depend on other tokens' scores. Softmax introduces competition between tokens.

Why MHA' over Linear? A linear layer considers only local (per-token) information. MHA' aggregates global context, which is essential for determining token redundancy.

Prefilling only: AG operates during the prefilling stage where the full sequence is available. It is not used during decoding to avoid slowing token-by-token generation.

Complexity Analysis

FLOPsAG = O(n2 d'k h')
FLOPsMHA after AG = O((1 - t%) n2 dk h)
FLOPsCombined = O(n2 max(d'kh', (1-t%)dkh))

Since h' << h and d'k < dk, AG overhead is small while the eviction savings are large.

Training Implementation

Two Training Modes

Continual Pre-Training (CPT)

Dataset: ~5,000 samples from RedPajama.
Hyperparams: τ=0.5, γ=2, α=5, β=0.4.
Training: Single epoch. Only 4 NVIDIA A100 GPUs required.
Base Model: LLaMA2-7B.

Supervised Fine-Tuning (SFT)

Datasets: PIQA, ARC-C, RTE, COPA, BoolQ, OpenBookQA.
Hyperparams: τ=0.3, γ=0, α=1 or 0.5, β=0.28.
Training: AdamW, lr=5e-5, 2 epochs per dataset.
LoRA: Wq, Wk, Wv, Wo + AG trainable.

Eviction Loss

ℓevict = α · |AG̅ - β|

Where AG̅ is the average AG output across all modules. The loss encourages the model to maintain eviction close to target ratio β, working alongside standard autoregressive language modeling loss.

Handling Non-Differentiability

The hard threshold in the gate G produces binary outputs that block gradient flow. AG uses the Straight-Through Estimator (STE): during the forward pass, hard 0/1 values are used; during backpropagation, the smooth Sigmoid output provides gradients.

Python (Conceptual)
# Straight-Through Estimator for AG gating
def attention_gate_forward(scores, tau, gamma):
    # Add initialization bias for early stability
    sigmoid_out = torch.sigmoid(scores + gamma)  # gamma >= 0

    # Hard threshold (forward pass)
    hard_flags = (sigmoid_out > tau).float()

    # STE: use hard values but pass gradients through sigmoid
    flags = hard_flags - sigmoid_out.detach() + sigmoid_out

    return flags  # {0, 1}^(n x h)

Code Walkthrough

The implementation modifies HuggingFace's LLaMA model by inserting AG modules before each MHA layer. The key file is modeling_llama_with_ag.py.

Python -- AG Integration
# Attention-Gate is applied before each MHA layer
# In the modified LlamaAttention forward:

class LlamaAttentionWithAG(nn.Module):
    def __init__(self, config):
        super().__init__()
        # Standard MHA components
        self.q_proj = nn.Linear(config.hidden_size, config.num_heads * self.head_dim)
        self.k_proj = nn.Linear(config.hidden_size, config.num_kv_heads * self.head_dim)
        self.v_proj = nn.Linear(config.hidden_size, config.num_kv_heads * self.head_dim)

        # Attention-Gate: small MHA' with h' << h heads
        self.ag_module = AttentionGate(
            hidden_size=config.hidden_size,
            num_ag_heads=4,       # h' = 4 vs h = 32
            head_dim=128,          # d'_k = 128
            tau=0.5,               # threshold
            gamma=2.0,             # init bias
        )

    def forward(self, hidden_states, attention_mask, ...):
        # Step 1: Generate eviction flags via AG
        eviction_flags = self.ag_module(hidden_states)  # (B, n, h)

        # Step 2: Build augmented attention mask
        # Combine causal mask + eviction flags
        ag_mask = build_ag_mask(eviction_flags, attention_mask)

        # Step 3: Standard MHA with masked KV-Cache
        attn_output = self.standard_mha(hidden_states, ag_mask)
        return attn_output

Python -- Mask Construction
def build_ag_mask(eviction_flags, causal_mask):
    """
    Construct attention mask with AG eviction.

    For each head i, position (j, t):
      m_i^(j,t) = 0         if j < t   (causal)
      m_i^(j,t) = 1         if j = t   (self-attention preserved)
      m_i^(j,t) = AG(X)_i^t  otherwise  (eviction decision)
    """
    bsz, seq_len, num_heads = eviction_flags.shape

    # Start with causal mask
    mask = causal_mask.clone()

    # Apply eviction flags (columns masked for evicted tokens)
    for head_idx in range(num_heads):
        evict_mask = eviction_flags[:, :, head_idx]  # (B, n)
        mask[:, head_idx, :, :] *= evict_mask.unsqueeze(2)

    # Always preserve diagonal (self-attention)
    diag_indices = torch.arange(seq_len)
    mask[:, :, diag_indices, diag_indices] = 1

    return mask

Key Implementation Patterns

The source code follows HuggingFace Transformers conventions. The AG module is implemented as a subclass that hooks into the standard LLaMA decoder layer. Key patterns include:

Python -- AG Module Structure
class AttentionGate(nn.Module):
    """
    Lightweight gating module placed before each MHA layer.
    Produces binary retain/discard flags per token per head.

    Architecture:
      1. MHA' (reduced heads): global information exchange
      2. Linear projection: h'd'_k -> h (one score per MHA head)
      3. Sigmoid + threshold: hard binary decision
      4. STE: gradient approximation during backward pass
    """
    def __init__(self, hidden_size, num_ag_heads, head_dim, tau, gamma):
        super().__init__()
        self.mha_prime = MultiHeadAttention(
            hidden_size=hidden_size,
            num_heads=num_ag_heads,   # h' = 4 typically
            head_dim=head_dim,        # d'_k = 128
        )
        self.gate_proj = nn.Linear(num_ag_heads * head_dim, num_mha_heads)
        self.tau = tau
        self.gamma = gamma

    def forward(self, hidden_states):
        # Step 1: Global context via mini MHA
        context = self.mha_prime(hidden_states)   # (B, n, h'*d'_k)
        # Step 2: Project to per-head scores
        scores = self.gate_proj(context)          # (B, n, h)
        # Step 3: Sigmoid gating with STE
        probs = torch.sigmoid(scores + self.gamma)
        hard = (probs > self.tau).float()
        flags = hard - probs.detach() + probs     # STE trick
        return flags                              # (B, n, h)

Design Rationale & Analysis

Why Global Information Matters

The fundamental question in KV-cache eviction is: which tokens are redundant? This question cannot be answered by examining each token in isolation. A token that seems uninformative locally may be critical when viewed in the context of the entire sequence. Consider a pronoun "it" -- its importance depends entirely on what "it" refers to, which can be arbitrarily far away in the context.

AG addresses this by using MHA' (a mini multi-head attention module) to aggregate information across the entire sequence before making eviction decisions. This is in contrast to local methods that rely on per-token hidden states or local windows of attention scores.

Empirical Evidence: When MHA' is replaced with a simple linear layer (Setting 5 in the ablation), the eviction ratio drops from 60% to 0.86% -- the linear layer essentially learns to retain everything because it cannot determine token importance from local information alone.

Layer-Wise Behavior

The visualization of AG patterns reveals a consistent and interpretable behavior:

Early Layers (0-5)

Dense attention patterns. AG retains nearly all tokens. These layers build foundational representations and need most of the context available.

Middle Layers (6-20)

Progressive sparsification. AG learns to increasingly evict redundant tokens. The information has been distilled into key representations.

Deep Layers (21-31)

Highly sparse. Some heads in deep layers may be entirely masked (all tokens evicted except self). This aligns with findings that deep layers perform narrower, more specific computations.

Heavy Hitter Token Analysis

AG's visualization reveals that certain tokens -- called "Heavy Hitters" in H2O terminology -- appear as bright vertical lines across multiple heads and layers. These are tokens that consistently receive high attention scores. AG naturally learns to preserve these Heavy Hitters while evicting the less important surrounding tokens, validating the Heavy Hitter hypothesis from a learned perspective rather than relying on accumulative score heuristics.

Limitations & Future Work

Prefilling-Only

AG currently operates only during prefilling. Extending it to the decoding stage for dynamic eviction during generation is left as future work. The challenge is that AG itself uses attention, so running it at each decode step adds latency.

For-Loop Over Heads

The current implementation iterates over attention heads with a Python for-loop during prefilling, which is suboptimal. Kernel fusion techniques could parallelize this for better wall-clock time.

Efficiency at Deeper Layers

Since deeper layers evict more aggressively, the AG module at deep layers could potentially use even fewer heads or smaller dimensions. This progressive compression of the AG itself could further reduce overhead.

Initialization Strategy: Xavier + Bias

The AG parameters are initialized using Xavier initialization (Glorot & Bengio, 2010) for weight stability. Additionally, a small positive constant γ ≥ 0 is added inside the Sigmoid function to ensure initial retention probabilities are close to 1. This means the model starts by retaining most tokens and gradually learns which to evict during training.

G(s, τ) = 1  if  Sigmoid(s + γ) > τ,  else  0

With γ = 2 and τ = 0.5, the initial Sigmoid(0 + 2) = 0.88 > 0.5, so most tokens are retained initially. As training progresses, the model learns to produce negative scores for redundant tokens, pushing them below the threshold.

Position in the KV Eviction Landscape

Attention-Gate occupies a unique position in the KV cache eviction research landscape. It is one of the few methods that introduces trainable parameters for eviction decisions, sitting between training-free heuristics and fully-trained sparse attention methods:

Training-Free Methods

StreamingLLM, H2O, NACL, A2SF: No additional parameters. Fast to deploy but limited adaptability. Cannot learn task-specific or model-specific eviction patterns.

Lightweight Learnable (AG, LookaheadKV)

Attention-Gate, LookaheadKV: Small additional parameters (<1% of model). Can learn eviction patterns while keeping the base model frozen. Best balance of flexibility and efficiency.

Full Architecture Changes

GQA, MQA, Sparse Transformers: Require retraining the entire model. Maximum flexibility but highest cost. Not applicable to existing pre-trained models.

10. Implementation & Integration Guide

Attention-Gate adds fewer than 1% parameters and integrates cleanly with existing attention implementations as a per-head linear gate module.

Integration StepDescription

3. Fine-tuneTrain gates on task-specific data (1-5% of full training budget)
4. Prune at thresholdZero out heads with gate value below threshold τ (typically 0.1)
Tip: For KV cache compression, focus training on long-context tasks. Gates learn which heads are critical for long-distance contextual coherence.
When to Use Attention-Gate: Attention-Gate excels when: task-specific fine-tuning is feasible, long-context inputs (>4K tokens) are the norm, and memory is the bottleneck. For general chatbots, simpler streaming eviction may suffice.

Composability

Stack with 4-bit KV quantization for multiplicative compression: gate prunes heads × quantization reduces precision = 8-16× total reduction.

Training cost

Gate params <1% of model. Fine-tuning takes ~10% of full pre-training compute — one-time cost per deployment target.
