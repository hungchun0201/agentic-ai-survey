3 AgentInfer Framework

AgentInfer decomposes agent optimization into four cooperative modules, each targeting a specific latency or inefficiency source:

Stage 1: AgentCollab

Hierarchical dual-model collaboration with self-evaluation-driven escalation. Small model handles routine steps; large model rescues stalled trajectories.

Reasoning Level

Stage 2: AgentCompress

Semantic compression that filters search results and asynchronously distills environment memory while preserving reasoning traces.

Context Optimization

Stage 3: AgentSched

KV-cache-aware hybrid scheduler using a shadow-price mechanism to dynamically balance SJF responsiveness with cache preservation.

System Level

Stage 4: AgentSAM

Cross-session speculative decoding via suffix automaton with context memory. Accelerates token generation by reusing patterns from prior sessions.

5 AgentCompress: Semantic Context Compression

Agent context bloats rapidly as search results accumulate, often exceeding 80K tokens. AgentCompress addresses this with two complementary modules:

Search-Enhanced Compression

After each batch web search, a lightweight model filters and ranks results in 5-10 seconds, pruning ~20% of irrelevant search tokens before they enter the url_crawler and document_qa stages.

Result: +3.50% accuracy, 11.2% fewer url_crawler calls, 38% fewer document_qa calls.

Asynchronous Context Distillation

Separates memory into reasoning memory (preserved) and environment memory (distilled). When context exceeds 5K tokens, a background task asynchronously summarizes environment data without blocking the agent's reasoning.

Key finding: Removing reasoning memory caused a 1.8x latency increase due to lost situational awareness.

6 AgentSched: KV-Aware Hybrid Scheduler

AgentSched uses a shadow-price (lambda) mechanism to dynamically blend SJF and KV-cache-preserving scheduling. When KV capacity is loose, it prioritizes short jobs; when tight, it favors requests with cached context.

// Shadow price computation
U = max(0, N - H)         // Effective usable KV capacity
D = sum(need_i)            // Total new demand
z = D / (U + epsilon)
lambda = lambda_max * sigmoid(k * (z - 1))

// Per-request scoring
sjf_mix = clamp(1 - lambda/lambda_max, 0, 1)
need_eff = sjf_mix * need_tok + (1 - sjf_mix) * need_blocks
score_i = a * hit_i - (b + lambda) * need_eff_i + c * wait_i

9 Agent Workflow Analysis

The Think-Act-Observe Execution Loop

AgentInfer focuses on Deep Research Agents that operate in a persistent Think-Act-Observe cycle. At each iteration:

  1. Think: The agent interprets its goal and contextual history, formulates a plan. Triggers one or more LLM inference calls.
  2. Act: The agent invokes external tools -- web_search, python("code"), knowledge-base retrieval.
  3. Observe: Tool output is integrated into the evolving context for subsequent reasoning.

Multi-Agent Orchestration: DeepDiverV2

In multi-agent frameworks like DeepDiverV2, the central Planner Agent decomposes tasks into sub-tasks for specialized sub-agents:

Two Fundamental Bottlenecks

Inference Latency

Each Think phase triggers synchronous LLM inference with token-by-token decoding. As the number of reasoning steps grows, accumulated inference delay becomes the dominant contributor to wall-clock time. Multi-agent setups amplify this through concurrent reasoning calls competing for GPU resources.

Context Explosion

Every iteration appends reasoning traces, tool invocations, and observations. Prompt length grows linearly with turns, but self-attention cost grows quadratically. Without memory management, context can exceed 80K tokens, degrading attention precision and forcing eventual truncation.

10 AgentCompress: Detailed Design

Why Retaining Reasoning Memory is Critical

AgentCompress separates context memory into two components:

  • Reasoning memory (Think + Act steps): The agent's chain of thought and planned actions. Must be preserved.
  • Environment memory (tool results, search content): Raw data from external tools. Can be distilled.
Critical Finding: Cognitive Confusion from Over-Compression

When only compressed context memory was kept (reasoning memory removed), the agent lost awareness of its trajectory and current progress. It failed to stop search loops autonomously, doubling iterations and causing a 1.8x increase in end-to-end latency.

Asynchronous KV Pre-computation

After context compression, the reconstructed context has a different prefix from the original, causing a cache miss. To avoid expensive re-computation, HCache pre-computes the new context's KV cache asynchronously in a background thread after compression completes (typically after T1). The new KV cache takes effect at T2, meaning the agent processes through 2-3 additional rounds before the compressed context is adopted -- a natural delay that avoids any blocking.

11 AgentSched: Shadow-Price Mechanism Deep Dive

Why FCFS and SJF Both Fail for Agents

FCFS Problems

Long, compute-heavy planner requests block numerous short, latency-sensitive tool agent requests (document_qa, url_crawl). This creates head-of-line blocking that stalls the entire agent workflow.

SJF Problems

Pure SJF consistently deprioritizes long-running agents, causing starvation. More critically, it ignores KV cache state: scheduling a short, cache-cold request can evict a long agent's cached context, triggering a costly 3.1-second recomputation upon resumption.

The Shadow-Price Controller

AgentSched monitors two key quantities continuously:

  • Effective Usable Capacity (U): Free KV blocks minus those reserved for cache-hit requests
  • Total New Demand (D): Sum of new KV blocks needed by all queued requests

The shadow price lambda = lambda_max * sigmoid(k * (D/U - 1)) provides a smooth, continuous control signal that transitions between SJF mode (lambda near 0, loose KV capacity) and KV-preserving mode (lambda near lambda_max, tight capacity).

Score Function Breakdown

score_i = a * hit_i              // Reward for KV cache hit
         - (b + lambda) * need_eff_i  // Penalty for new KV demand
         + c * wait_i              // Fairness: prevent starvation

// Where need_eff blends token-based and block-based need:
sjf_mix = clamp(1 - lambda/lambda_max, 0, 1)
need_eff = sjf_mix * need_tok + (1 - sjf_mix) * need_blocks

When lambda is near 0 (loose cache), the score favors short jobs (SJF behavior). When lambda is high (tight cache), it increasingly penalizes cache-cold requests and rewards cache-warm ones, preserving valuable context for long-running agents.

Design Philosophy

AgentInfer demonstrates that achieving true efficiency in autonomous agents requires a fundamental shift from micro-optimizing isolated components to macro-optimizing the end-to-end task workflow. Each module targets a specific bottleneck (reasoning redundancy, context bloat, cache thrashing, token generation speed), and their compositional gains prove that the whole is greater than the sum of its parts.

13 Case Study: Trajectory Comparison

A detailed trajectory comparison across three modes (small-model-only, AgentCollab, large-model-only) on a BrowseComp-zh task illustrates the value of self-evaluation-driven collaboration:

Small Model Only

The agent falls into local search loops, repeatedly exploring variants of the same query (e.g., "Cairo Declaration"). It fails to broaden its search strategy and misses the key target document entirely. Yields the lowest final quality.

AgentCollab

The small model initially follows a similar path. When Progress Check returns FALSE (stagnation detected), control escalates to the large model, which broadens the search space and recovers the crucial evidence. After progress resumes (TRUE), control returns to the small model for remaining summarization. Matches large-model quality at lower cost.

Large Model Only

Maintains maximal coverage and consistently strong reasoning throughout, but at highest computational cost. The trajectory is similar to AgentCollab's final output quality.

Key Insight from Case Study

AgentCollab's self-evaluation correctly identifies stagnation as "no significant progress" and triggers escalation at exactly the right moment. The large model's broader search capability quickly unblocks the trajectory. This demonstrates that difficulty-aware escalation can match large-model quality while using the large model for only a fraction of the reasoning steps.

AgentSAM: Speculative Hit Rate Analysis

Both OTE (Overall Token Efficiency) and SHR (Speculative Hit Rate) improve with longer prompts, confirming that longer contexts provide more repeated subsequences for the suffix automaton to exploit. By integrating cross-session memory, AgentSAM consistently outperforms vanilla SAM decoding at all prompt lengths tested (4K to 48K tokens).

Adaptive Speculative Decoding Switch

The runtime monitors context length and batch size. When these exceed empirically determined thresholds (where speculation overhead exceeds benefit), speculative decoding is automatically disabled, falling back to standard autoregressive generation. This prevents wasted validation effort in regimes where pattern repetition is unlikely.

14 Broader Impact and Future Directions

AgentInfer represents a paradigm shift in how agent systems are optimized. Rather than treating the LLM as a black box and optimizing only per-token throughput, it recognizes that agent efficiency is fundamentally about task completion time.

For Agent Framework Developers

AgentCollab's self-evaluation-driven escalation provides a template for hierarchical reasoning that can be applied to any multi-model agent framework. The Progress Check protocol is simple to implement and model-agnostic.

For Serving System Developers

AgentSched's shadow-price mechanism demonstrates that KV-cache-aware scheduling is essential for agentic workloads. Standard SJF and FCFS are both inadequate when workloads mix short tool calls with long reasoning chains.

For Compression Researchers

AgentCompress shows that selective compression (preserving reasoning traces, distilling environment data) is crucial. Naive summarization causes cognitive confusion that increases total cost, not decreases it.

Future Work

Extensions include cross-agent speculative decoding (using Planner's output to predict Executor's response), predictive scheduling (anticipating tool call patterns), and integration with distributed KV cache systems for multi-node agent deployment.
