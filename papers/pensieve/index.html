Overview

The Redundant Recomputation Problem

Existing stateless serving systems (vLLM, TensorRT-LLM) prepend the entire conversation history to each new request and re-process it from scratch. As conversations grow, prefill cost dominates: for 32 requests with 2048-token history, prefill takes 4x longer than 200 steps of generation. Most of this computation is redundant—the same history tokens are processed repeatedly.

Pensieve: Make Serving Stateful

Pensieve saves a conversation's processed context (KV-tokens) in a multi-tier GPU-CPU cache and reuses it across follow-up requests. Only new user prompt tokens need processing. Key challenges addressed: limited GPU memory, token-level eviction between tiers, and handling non-contiguous KV cache scattered across GPU memory.

Key Mechanisms

Multi-Tier GPU-CPU Cache

GPU KV cache serves as both workspace and storage. CPU memory extends capacity. Token-level granularity (32-token chunks) enables fine-grained eviction. Earlier tokens evicted first (cheaper to recompute due to causal attention). Retention value V = Cost(s,l) / T accounts for recomputation cost and inactivity time.

Ahead-of-Time Swapping

Proactive swap-out when GPU free slots fall below 25%. Transfer overlaps with computation, hiding PCIe latency. Prioritizes data retrieval over eviction to avoid contention on duplex PCIe bus (18-20% throughput drop observed).

Pipelined KV Recovery

Layer-by-layer swap-in pipelined with model computation. Layer L+1 transfer overlaps with layer L attention. GPU events coordinate ordering. Dropped tokens handled via recomputation, split into two sub-requests.

Multi-Token Attention Kernel

Novel GPU kernel built on NVIDIA Cutlass. Handles multiple query tokens per request over non-contiguous KV cache. Uses matrix-matrix multiplication (not matrix-vector like PagedAttention). Supports causal masking for ragged batch sizes. Matches ideal performance of contiguous memory.

Unified Batch Scheduling

Combines prefill and generation requests in the same batch. Single-token generation is a special case of multi-token attention with query size 1. Avoids separate small-kernel invocations, improving GPU utilization. The scheduler is clocked by generation step completions with FCFS admission policy.

Significance

Stateful Serving Paradigm

Establishes that LLM serving should be stateful for multi-turn workloads. Eliminates the dominant cost: redundant history recomputation across conversation turns.

Novel Attention Kernel

Multi-token attention over non-contiguous KV cache is a reusable building block, enabling efficient prefill with scattered cached context.

Cost-Aware Eviction

Eviction accounts for recomputation cost (causal attention makes earlier tokens cheaper). 4.4% higher CPU cache hit rate and 14.6% fewer recomputed tokens vs LRU.

Real-World Validation

Evaluated on ShareGPT (real user conversations) and UltraChat. Benefits persist across user think times from 30 to 600 seconds, demonstrating robust practical value.
