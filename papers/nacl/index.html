Overview

The Problem: Biased Eviction Scores in Long-Context Inference

Hosting LLMs with extended context windows is cost-prohibitive due to the O(l * n_h * d_h) memory consumption of KV caches. A 7B model with batch size 4 at 32K sequence length requires 64GB of KV cache -- 4.7x larger than the model weights.

Existing eviction methods (H2O, Attention Sink, MSRNN, Scissorhands) rely on accumulated or local attention scores to determine token importance. These scores suffer from attention bias: recent tokens receive disproportionately high attention scores, while mid-context tokens are undervalued. Furthermore, the attention score distribution becomes progressively flattened with increasing text length, making score-based methods less capable of identifying truly important tokens.

These biased methods report good perplexity numbers but fail on real generation tasks, especially long-context tasks where pivotal information may appear anywhere in the input.

NACL: A Two-Part Hybrid Solution

NACL (a general and effective KV Cache eviction framework for LLMs at inference time) proposes a hybrid eviction strategy that combines two complementary mechanisms:

  1. Proxy-Tokens Eviction: Uses a small subset of task-relevant proxy tokens (typically from the user's question) to compute unbiased importance scores via global attention statistics
  2. Random Eviction: Head-wise random sampling from the attention score distribution with different seeds per head and layer, providing probabilistic information preservation guarantees

NACL achieves 80% and 75% performance improvement on short- and long-text tasks respectively over existing methods with 50% KV cache reduction, and 5x total KV reduction with over 95% performance maintenance.

Key Observations Motivating NACL

Observation 1: Evaluation Metrics Are Misleading

Current eviction methods are evaluated primarily using perplexity (PPL) on short texts. PPL emphasizes textual fluency but does not capture the ability to generate correct answers for comprehension tasks. Methods like Attention Sink claim infinite-length support but fail on tasks requiring the model to actually generate from long-context information (see Table 1 in the paper).

NACL re-evaluates all methods using both lm-eval-harness (few-shot reasoning) and LongBench (generation from long documents), providing a more honest assessment.

Observation 2: Attention Bias Corrupts Scoring

Visualizing attention score distributions reveals the attention bias problem: at each generation step, tokens directly preceding the current token receive disproportionately high attention, while earlier tokens are comparatively diminished. This leads to:

  • H2O overemphasizes initial and recent tokens, creating a triangular retained pattern
  • MSRNN uses only the current token's attention, neglecting global context
  • Both methods fail to retain pivotal mid-context tokens (e.g., passkeys)

NACL's proxy-token approach uses a carefully selected subset to compute scores, avoiding this bias by focusing on task-relevant attention patterns.

Mechanism 1: Proxy-Tokens Eviction

The Proxy Tokens Hypothesis

Within the input prompt x_prompt, there exists a small subset of tokens P (the proxy tokens) whose attention patterns most precisely estimate the importance of all other tokens for the task at hand. In practice, the proxy tokens are typically the user's question tokens, which appear at the end of the prompt.

The scoring function uses only the proxy tokens' attention rows:

F_score(A, C) = Sum_{x_p in P} Softmax(A(x_p, *))

This produces a column-wise importance score for each token in the KV cache. The top-C_p tokens by this score are retained, along with the proxy tokens themselves (always protected).

Why Proxy Tokens Work

When computing attention for a question token x_p, the model naturally attends most strongly to the context tokens most relevant to answering that question. By summing the softmax-normalized attention scores across all proxy tokens, NACL obtains a task-specific importance signal that is:

  • Free from the recency bias that affects all-token accumulation
  • Focused on tokens relevant to the downstream task
  • Computed from global attention statistics (not greedy step-by-step)

Special Cases and Budget

The proxy token framework subsumes existing methods:

  • When P = all tokens: NACL reduces to H2O (accumulating all attention scores)
  • When P = {current token}: NACL reduces to MSRNN (using only current attention)
  • Optimal budget: ~10% of prompt tokens as proxy tokens

Setting the proxy budget too low (0%) loses important scoring context; too high (100%) reintroduces the redundant information that causes attention bias. The paper finds 10% strikes the optimal balance.

Eviction as optimization: NACL formulates the encoding-phase eviction as a combinatorial optimization problem: S_t = argmax_{S_t subset R} Sum_{x in S_t} F_score(A, C_p), subject to |S_t| = C_p, where R = x_prompt \ P. This global view outperforms greedy step-by-step eviction by 1.4 points on average.

Mechanism 2: Head-Wise Random Eviction

Diversified Randomness for Robustness

Pure score-based eviction, even with proxy tokens, can still miss important tokens whose attention signal is weak. NACL introduces a random eviction component that samples tokens from a score-weighted probability distribution, independently per attention head with different random seeds.

S_random ~ Softmax(F_score),   |S_random| = C_r

Each head n in each layer i uses a different random seed: seed = hash(layer_i, head_n). This ensures that different heads retain different subsets of tokens, creating a diversified coverage across the model.

Probabilistic Guarantee: Near-Certain Token Retention

For LLaMA-7B with 32 layers and 32 heads per layer:

  • At C = 20% budget: the probability of any token being retained in at least one head's KV cache is 99.92%
  • At C = 1% budget: the probability of retention in at least one layer is 1 - (C^h)^l = 1 - (0.01^32)^32, which is >99.99%

This means that even under aggressive eviction, virtually every token's information is preserved somewhere in the model, eliminating the catastrophic information loss that deterministic methods can cause.

Complete NACL Algorithm

# Algorithm 1: NACL Complete Pipeline def nacl_encoding(prompt, C_p, C_r, proxy_tokens P): """One-time global eviction during encoding phase""" for layer_i in model.layers: for head_n in layer_i.heads: # Compute full attention scores A = compute_attention(prompt, layer_i, head_n) # A in R^{p x p} # Proxy-Tokens scoring: use only proxy rows F_score = sum(softmax(A[x_p, :]) for x_p in P) # Deterministic eviction: keep top-C_p by score + protect proxy R = x_prompt \ P # candidate tokens u_score = top_k(F_score[R], C_p) | P # always retain proxy # Random eviction: sample C_r tokens from score distribution P_prompt = softmax(F_score) # probability dist u_random = sample(P_prompt, C_r, seed=hash(layer_i, head_n)) # Merge retained sets S_encoding[layer_i][head_n] = u_score | u_random return S_encoding def nacl_generation(S_encoding, max_length, m): """Incremental eviction every m steps during generation""" S = S_encoding for t in range(1, max_length): for layer_i in model.layers: for head_n in layer_i.heads: # Append new token's KV to cache K[layer_i][head_n] = concat(K_prev, z_t * W_K) V[layer_i][head_n] = concat(V_prev, z_t * W_V) # Periodic eviction using current token's attention if t % m == 0: A = compute_attention(z_t, layer_i, head_n) S[layer_i][head_n] = Eviction(A, C) # same NACL policy z_t = sample(model.predict(z_t))

Memory Reduction Comparison

At comparable accuracy to H2O (20% budget), NACL achieves the same performance with only 10% budget -- a 3x additional KV cache reduction. In absolute terms, NACL at 10% budget reduces the KV cache to 10% of original size (a 10x reduction) while maintaining 29.4 average accuracy vs H2O's 28.6 at 20%.

For LLaMA-7B with batch size 4 at 32K sequence length in bf16 precision: full cache = 62.5 GB, H2O (20%) = 12.5 GB, NACL (10%) = 6.25 GB.

Proxy Token Budget Sensitivity

The proxy token budget (as % of prompt tokens) shows clear sweet spot behavior:

  • 0% proxy: Degenerates to MSRNN -- insufficient scoring context
  • 10% proxy: Optimal -- captures task-relevant attention without redundancy
  • 100% proxy: Degenerates to H2O -- introduces redundant information that corrupts scoring

Performance peaks at ~10% and declines monotonically above that.

Random Budget Ratio

Increasing the random budget from 10% to 70% yields 2.25% to 8.17% average accuracy improvement over 0% random budget. Performance peaks at 70% random ratio. Beyond 90%, pure randomness without score-based selection causes decline.

The balance between deterministic scoring (Proxy-Tokens) and stochastic diversity (Random) is essential: neither alone matches their combination.

Sampling distribution choice: Sampling based on global statistical attention scores (softmax of F_score) outperforms uniform random sampling. The attention scores provide an informative reference for randomness, biasing the sampling toward more likely-important tokens while maintaining diversity. This score-guided randomness is a key design choice.

Analysis

KV Cache Memory Scaling

At comparable accuracy to H2O (20% budget), NACL (10% budget) reduces the linearly-growing KV cache to 10% of original size across all sequence lengths. At 32K tokens, this means 6.25 GB instead of 62.5 GB -- fitting comfortably on a single A100 40GB GPU that would otherwise be memory-limited.

The memory reduction is more significant for longer sequences: at 1K tokens, all methods perform similarly; at 32K, the gap between NACL and baselines widens substantially.

Interpretable Eviction Patterns

Visualizing the attention score matrices after eviction reveals distinct patterns:

  • H2O: Triangular pattern -- retains only initial and recent tokens
  • MSRNN: Similar triangular bias with slight variation
  • NACL: More uniform coverage -- proxy tokens guide sampling toward mid-context, while head-wise randomness fills gaps. Initial and recent tokens are still protected, but critical mid-context information is also preserved.

Perplexity vs Generation Quality

An important finding: all eviction methods achieve comparable log-PPL at 20% budget (H2O: 6.4, MSRNN: 4.0, NACL: 4.0 on 5-shot; ~3.2-3.3 on 25-shot). However, their generation quality differs drastically (35.9 vs 60.3 vs 63.8 average accuracy). This confirms the paper's argument that perplexity is an unreliable metric for evaluating KV cache eviction methods, especially for long-context tasks. Generation-based benchmarks like LongBench are essential for honest evaluation.

Significance and Limitations

Training-Free, Plug-and-Play

NACL requires no retraining, fine-tuning, or model modification. It operates purely at inference time, making it immediately applicable to any decoder-only LLM in production. The approach is compatible with FlashAttention-2 via a custom CUDA kernel.

Principled Randomness with Guarantees

Head-wise random eviction provides mathematical probabilistic guarantees: >99.92% token retention at 20% budget, >99.99% at 1%. This is the first eviction method to offer formal preservation bounds through diversified sampling.

Unifying Framework

NACL subsumes H2O and MSRNN as special cases (100% and 0% proxy budgets respectively). It provides a principled design space with clear configuration knobs: proxy budget, random budget, eviction interval m, and sampling distribution.

Honest Evaluation Standard

The paper establishes that perplexity is insufficient for evaluating eviction methods and introduces evaluation on both short-text (lm-eval-harness, 7 tasks) and long-text (LongBench, 7 tasks) generation benchmarks. This sets a higher bar for future work.

Limitations: (1) Due to resource constraints, NACL has not been extensively tested across various large-scale models (70B+) or ultra-long contexts (128K+). (2) The selection of proxy tokens currently relies on heuristics (choosing question tokens at the end of the prompt). Automatically and adaptively determining optimal proxy tokens from the model itself remains an open challenge. (3) The eviction interval m during generation is a fixed hyperparameter; adaptive eviction scheduling could further improve efficiency.

Related Work and Context

Prior Eviction Methods

Attention Sink (Xiao et al. 2023): Keeps initial tokens (attention sinks) plus recent tokens, enabling infinite-length processing. However, it completely discards mid-context information, failing catastrophically on tasks that require comprehension beyond the recent window.

H2O (Zhang et al. 2023): Uses the summation of accumulated attention scores for greedy eviction, retaining a balance of recent and "heavy hitter" tokens. While achieving fair results, H2O's reliance on biased local statistics leads to 4.3-point degradation on short tasks and 2.9 on long tasks.

MSRNN (Oren et al. 2024): Uses only the current token's attention score for eviction decisions. This avoids accumulation bias but neglects global context, resulting in similar performance to H2O.

Scissorhands (Liu et al. 2023): Tracks tokens whose attention scores fall above average within a history window. Performance is sensitive to window size and struggles with long-context tasks where pivotal tokens may be distant.

Complementary Approaches

Architecture Modification: MQA (Shazeer 2019), GQA (Ainslie et al. 2023), and MLA (DeepSeek-AI 2024) reduce KV cache at the architecture level. These require retraining and are complementary to NACL, which operates at inference time without model changes.

Learnable Eviction: Anagnostidis et al. (2023) propose learning which tokens to prune, but this requires additional training. NACL achieves comparable results training-free.

Length Extrapolation: Position interpolation (Chen et al. 2023), Attention Sink (Xiao et al. 2023), and LM-Infinite (Han et al. 2023) extend context windows but do not address memory footprint. NACL directly reduces KV cache size.

NACL as a Unifying Framework

NACL provides a principled design space that encompasses existing methods as special cases and introduces new mechanisms (proxy-token scoring, head-wise randomness, global one-time eviction) that consistently outperform prior art. The framework offers clear configuration knobs:

Implementation Details

Experimental Setup

All experiments conducted on a single NVIDIA A100 (80GB) GPU using representative open-sourced LLMs: LLaMA2-7B-base and LLaMA2-7B-Chat (Touvron et al. 2023). Results averaged over various seeds for reliability.

Short-text evaluation: Seven tasks from lm-eval-harness (Gao et al. 2021): PiQA, COPA, OpenBookQA, Winogrande, SciQ, ARC-E, and ARC-C. Perplexity reported on OpenBookQA for generalization assessment.

Long-text evaluation: Seven tasks from LongBench (Bai et al. 2023): PassageRetrieval-Zh, PassageRetrieval-En, RepoBench-P, HotpotQA, NarrativeQA, TriviaQA, and QMSum. Context window set to 4K length.

FlashAttention-2 Integration

NACL's custom CUDA kernel recovers attention scores from FlashAttention-2's log-sum-exp (LSE) output. FA2 does not materialize the full attention matrix during forward pass; instead, it outputs the LSE values that can be used to reconstruct the necessary scoring information. This makes NACL compatible with memory-efficient attention implementations used in production, supporting 128K context with only 15GB memory at 20% eviction rate.

The kernel computes the proxy-token subset of attention scores using the LSE values, avoiding the O(n^2) memory cost of materializing the full attention matrix.

Proxy Token Selection in Practice

The proxy tokens P are selected from the end of the input prompt, where the user's question or instruction typically resides. This heuristic works well because:

  • In conversational settings (LLaMA2-Chat), the user's latest question directly indicates what information is task-relevant
  • In RAG settings, the query appended after the retrieved context focuses attention on the most important passages
  • In few-shot learning, the final example and question are the most informative proxy for scoring

The choice of proxy tokens can be flexibly adapted to various application scenarios based on task orientation. For instance, in summarization tasks, the instruction tokens ("Summarize the following document:") serve as effective proxies.

Future Directions

Adaptive Proxy Selection

Automatically identifying optimal proxy tokens from the model's internal representations, rather than relying on position-based heuristics, could further improve eviction quality. Potential approaches include using the model's own attention patterns in early layers to identify task-critical tokens, or training a lightweight classifier to select proxies.

Ultra-Long Context Support

Testing NACL on 128K+ context models (GPT-4 class, Llama3-128K) would validate its scalability. At these lengths, the attention bias problem is even more severe, and NACL's proxy-token + randomness combination should provide even larger advantages. The key challenge is efficient attention score computation at these scales.
