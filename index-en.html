S1

Workload Characterization

Agentic LLM workloads differ fundamentally from traditional chatbot traffic across five key dimensions: multi-turn depth (50–200+ inference calls per program vs 1–3 turns), tool-call-induced pauses that fragment GPU utilization, monotonically growing context that pressures KV cache capacity, redundant prefill across turns and agents, and CPU-dominated latency (tool execution time far exceeds LLM inference time). Understanding these patterns is a prerequisite for infrastructure co-design.

Continuum — KV Cache TTL for Agent Scheduling
arXiv'252511.02230repo Agentic ✓PD —
Provides the most detailed empirical analysis of agent traffic, using SWE-Bench and BFCL traces. Finds that most tool calls complete in <2 seconds and are highly predictable; scheduling bubbles accumulate to 58.2% of baseline end-to-end latency. Open-sources the first public agentic inference benchmark traces.
The Cost of Dynamic Reasoning
HPCA'262506.04301 Agentic ✓PD —
First comprehensive system-level analysis comparing four agent frameworks: ReAct, Reflexion, LATS, and LLMCompiler. Reveals an "imminent sustainability crisis": agents trade compute for accuracy, but exhibit sharply diminishing returns, widening latency variance, and order-of-magnitude cost increases.
A CPU-Centric Perspective on Agentic AI
arXiv'252511.00739repo Agentic ✓PD —
Profiles five representative workloads (Haystack RAG, Toolformer, ChemCrow, Langchain, SWE-Agent), revealing an overlooked bottleneck: CPU-side tool processing can consume up to 90.6% of total latency. Proposes CGAM and MAWS, achieving 2.1x P50 latency reduction on Intel Emerald Rapids + NVIDIA B200.
PrefillShare — Shared Prefill for Multi-LLM
arXiv'262602.12029 Agentic ~PD Disagg
Quantifies redundant prefill in multi-model agent pipelines (Planner->Coder->Reviewer) and proposes cross-model shared prefill modules for KV reuse.
AgentInfer — Co-Design of Inference Architecture
arXiv'252512.18337 Agentic ✓PD —
Decomposes agent latency into LLM calls, tool calls, retrieval, and environment interaction, proposing a co-design framework for inference architecture and systems.
S2

Prefill-Decode Disaggregation

Prefill-decode (PD) disaggregation -- assigning the compute-intensive prefill phase and memory-bandwidth-intensive decode phase to different GPU pools -- has become the dominant serving architecture by 2025. However, agentic workloads with their extremely high KV cache reuse rates (>=95%) shift the bottleneck from compute to KV cache I/O, requiring architectural rethinking.

DistServe — PD Disaggregation for Goodput
OSDI'242401.09670repo Agentic —PD DisaggDistributed
Establishes the foundation for PD disaggregation, independently optimizing resource allocation and parallelism strategies for each phase. 7.4x throughput or 12.6x tighter SLO. By 2025, virtually all production-grade systems have adopted this architecture. Limitation: FCFS scheduling, no multi-turn KV cache reuse.
Splitwise — Phase Splitting for LLM Inference
ISCA'242311.18677repoBest Paper Nominee Agentic —PD DisaggDistributed
Characterizes the hardware profiles of each phase: prefill is compute-bound, decode is memory-bandwidth-bound, with distinct power characteristics. Enables heterogeneous hardware deployment (H100 prefill + A100 decode), 1.4x throughput, 20% lower cost.
Mooncake — KVCache-centric Architecture
FAST'252407.00079repo★ Best Paper Agentic ~PD DisaggDistributed
Production-grade serving platform for Kimi (Moonshot AI), processing 100 billion+ tokens daily. Leverages underutilized CPU/DRAM/SSD to build a distributed KV cache pool; Transfer Engine achieves 87 GB/s over 4x200 Gbps RoCE. Integrated into vLLM, SGLang, TensorRT-LLM.
DualPath — Storage Bandwidth Bottleneck Fix
arXiv'262602.21548 Agentic ✓PD DisaggDistributed
The most directly relevant agentic disaggregation paper. Discovers that at KV cache hit rates >=95%, the prefill-side storage NIC saturates while the decode side idles. Proposes a new RDMA path from storage->decode->prefill, achieving 1.87x offline throughput, 1.96x online throughput. Challenges the compute-centric optimization view of the DistServe era.
TaiChi — Hybrid Aggregation-Disaggregation
arXiv'252508.01989 Agentic —PD DisaggDistributed
Rigorously proves that neither pure aggregation nor pure disaggregation is universally optimal. Proposes P-heavy/D-heavy GPU instances and a "latency transfer" strategy -- selectively degrading latency of SLO-met requests to help at-risk requests, achieving 77% goodput improvement.
NVIDIA Dynamo — Smart Router + KV Manager
GTC'25repo Agentic ✓PD DisaggDistributed
The most production-ready PD disaggregation framework with explicit agentic capabilities: Smart Router with radix-tree KV cache tracking, Planner for SLA-based dynamic P/D scaling, KV Block Manager for hierarchical offloading. v1.0 adds cache pinning TTL (nvext.cache_control) and LangChain integration. 30x throughput on GB300 NVL72.
TraCT — CXL Shared Memory KV Cache
arXiv'252512.18194 Agentic —PD DisaggDistributed
Uses CXL (Compute Express Link) shared memory as a network-free KV transfer substrate, enabling rack-scale KV cache sharing via direct load/store operations. 9.8x TTFT reduction, 6.2x P99 latency. CXL Type-3 devices are still in pre-production stage.
WindServe — Stream-based Dynamic Scheduling
ISCA'25 Agentic —PD DisaggDistributed
Stream-based disaggregation enabling asynchronous parallel execution within GPU instances. Dynamic prefill dispatch and stall-free rescheduling. 4.28x median TTFT, 1.5x TPOT P99.
NanoFlow — Nano-grained Pipeline Parallelism
OSDI'252408.12757repo★ Best Paper Agentic —PD —
Achieves near-optimal throughput via nano-grained pipeline parallelism. OSDI'25 Best Paper.
FlowKV — Low-Latency KV Transfer
arXiv'252504.03775 Agentic —PD DisaggDistributed
Achieves 96% KV transfer latency reduction through memory alignment, combined with load-aware scheduling.
PPD — Routing Append-Prefills to Decode Nodes
arXiv'262603.13358 Agentic ~PD Disagg
Routes append-prefills to decode nodes where the KV cache resides, reducing TTFT by 68% from the second turn onward.
DuetServe — Adaptive GPU Multiplexing
arXiv'252511.04791 Agentic —PD Disagg
Achieves intra-GPU disaggregation via SM-level spatial multiplexing.
BanaServe — Layer-level Weight Migration
arXiv'252510.13223 Agentic —PD DisaggDistributed
Layer-level weight migration, achieving 1.2–3.9x throughput on vLLM.
S3

KV Cache Management & Prefix Caching

KV cache management is the single optimization with the greatest impact on agentic workloads. The field has rapidly evolved from basic paged memory management to distributed, location-independent, workflow-aware caching systems.

vLLM — PagedAttention
SOSP'232309.06180repo Agentic —PD —
Introduces OS-inspired virtual memory and paging for KV cache, splitting tensors into fixed-size blocks with copy-on-write sharing. Near-zero memory waste, 2–4x throughput improvement. Establishes the foundation for all subsequent KV cache systems. Limitations: single GPU, no cross-request reuse, basic FCFS eviction.
SGLang — RadixAttention
NeurIPS'242312.07104repo Agentic ~PD —
Stores KV cache in a radix tree for automatic reuse across LLM calls, supporting few-shot, multi-turn, tree-of-thought, and agent template patterns. Cache-aware scheduling prioritizes requests with longer shared prefixes, achieving 5x throughput. Default LRU eviction is suboptimal for agentic workloads (future activations are predictable).
LMCache — Multi-tier KV Storage Layer
arXiv'252510.09665repo Agentic ~PD DisaggDistributed
First open-source KV caching layer supporting cross-query prefix reuse and PD disaggregation. Spans GPU HBM, CPU DRAM, disk, and remote storage (Redis, Mooncake), achieving 15x throughput. Pin API provides explicit retention control for agent sessions. Integrated into the vLLM production stack.
CachedAttention — Hierarchical KV Persistence
ATC'242403.19708 Agentic ~PD —
Implements hierarchical KV cache persistence (DRAM + disk) for multi-turn conversations, with position encoding decoupling and layer-level preloading. Achieves 86–90% cache hit rate on ShareGPT traces.
llm-d — K8s-native KV Cache Indexer
OSS'26repo Agentic ~PD DisaggDistributed
Kubernetes-native distributed inference by Red Hat/IBM/Google. Global KV-Cache Indexer with Precise Prefix-Cache Scorer, 87.4% cache hit rate, 88% faster TTFT. Session affinity prevents cache thrashing for multi-turn agents. v0.5 adds cache-aware LoRA routing.
CacheGen — KV Cache Compression Codec
SIGCOMM'242310.07240repo Agentic —PD —Distributed
Encodes KV cache tensors into compact bitstreams using a custom codec, exploiting distributional properties for 3.5–4.3x compression and 3.2–3.7x TTFT reduction. A key technique for transferring agent context across distributed nodes.
CacheBlend — Non-prefix KV Cache Reuse
EuroSys'252405.16444repo★ Best Paper Agentic —PD —
Enables reuse of precomputed KV caches for non-prefix text chunks by selectively recomputing 5–18% boundary tokens to recover cross-chunk attention. 2.2–3.3x TTFT, 2.8–5x throughput. Directly applicable to RAG agents that dynamically assemble multi-document context.
EPIC — Position-Independent Caching
ICML'252410.15332repo Agentic —PD —
Formalizes Position-Independent Caching using a compile-link analogy: chunks are "compiled" (independent KV cache) then "linked" (selective boundary recomputation). 8x TTFT, 7x throughput. Particularly valuable for agents where document ordering varies across invocations.
IC-Cache — Semantic In-context Caching
SOSP'252501.12689 Agentic —PD —Distributed
Semantic-level caching that uses historical request-response pairs as in-context examples to augment smaller models. Two-stage example selection and cost-aware routing, 1.4–5.9x throughput, 28–71% latency reduction. Exploits the observation that 70%+ of agent queries have semantically similar historical counterparts.
KVFlow — Workflow-aware Prefix Caching
NeurIPS'252507.07400 Agentic ✓PD —
First workflow-aware KV cache management framework. Abstracts agent execution as an Agent Step Graph, using "steps-to-execution" values to estimate temporal proximity of future activations, replacing LRU with workflow-aware priorities. Fully overlapped CPU->GPU prefetch eliminates cache miss stalls. 1.83x (single workflow) and 2.19x (parallel workflows) speedup.
Marconi — Prefix Caching for Hybrid LLMs
MLSys'252411.19379repo★HM Outstanding Paper Agentic ~PD —
First prefix caching system designed for hybrid LLMs (Attention + SSM). Admission/eviction policies predict reuse likelihood and weigh compute savings vs memory footprint, 34.4x token hit rate. Evaluated on SWE-Agent/SWE-Bench agentic workloads.
DiffKV — Differentiated KV Memory
SOSP'252412.03131repo★ Best Paper Agentic —PD —
Differentiates memory management for Keys and Values (different access patterns) with parallel KV compaction. SOSP'25 Best Paper.
Jenga — Heterogeneous KV Embeddings
SOSP'252503.18292repo Agentic —PD —
Handles heterogeneous KV embeddings in MLA-style models. Merged into vLLM mainline.
Pie — Programmable Serving System
SOSP'252510.24051repo Agentic —PD —
Programmable serving system enabling custom per-application KV cache policies via "inferlets". Yale.
DeepSeek MLA — Multi-Head Latent Attention
arXiv'242405.04434repo Agentic —PD —
Compresses KV tensors to a low-dimensional latent space, reducing KV cache by 93.3%. Used in DeepSeek-V2/V3/R1. Critical for maximizing concurrent agent sessions.
MHA2MLA — Retrofitting MLA
ACL'252502.14837repo Agentic —PD —
Enables existing MHA/GQA models to be retrofitted to MLA without retraining from scratch, via partial-RoPE removal and joint SVD approximation.
Oaken — Hybrid KV Cache Quantization
ISCA'252503.18599repo Agentic —PD —
Hybrid online-offline KV cache quantization, ISCA'25.
TokenLake — Unified Segment Prefix Cache
arXiv'252508.17219 Agentic —PD —Distributed
Aggregates GPU memory into a unified segment-level prefix cache pool for fine-grained elastic long-context serving.
S4

KV Cache TTL & Retention Policies

Cache retention -- deciding how long to keep KV caches in GPU memory during tool-call pauses -- is arguably the highest-leverage optimization for agentic workloads, yet remains the least studied area. The core tension: retaining cache avoids expensive re-prefill but blocks GPU memory for other requests; premature eviction wastes the prefill investment.

Open Problems: (1) No adaptive TTL -- values are currently set at request time and never updated; (2) No cross-program interaction modeling; (3) No semantics-aware retention; (4) No distributed TTL coordination; (5) No principled exploration-exploitation framework.
InferCept — KV Cache Preserve for Tool Calls
ICML'242402.01869repo Agentic ✓PD —
Introduces the simpler "preserve" operation, pinning KV cache between tool calls. Subsequent work (Continuum, KVFlow, ThunderAgent) identifies a key limitation: static pin policies fail under high-variance tool calls -- long or failed calls block GPU memory indefinitely.
Concur — AIMD Agent Admission Control
arXiv'262601.22705 Agentic ✓PD —
Identifies "middle-phase thrashing" in batch agentic inference: long-lived agents see cache efficiency collapse under concurrency pressure. Proposes AIMD-based agent admission control inspired by network congestion control, outperforming Continuum under high concurrency.
ThunderAgent — Program-aware Pause/Restore
arXiv'262602.13692repo Agentic ✓PD —
Program-aware inference with periodic memory monitoring and pause/restore operations using hysteresis watermarks. Proactively pauses active agents when KV cache thrashing is imminent. Outperforms vLLM and Continuum by 2.43–3.56x on workflows with predictable tool-call times.
SideQuest — Model-driven KV Eviction
arXiv'262602.22603 Agentic ✓PD —
Takes a fundamentally different approach: uses the reasoning model itself to decide which KV entries to evict via auxiliary "side quest" tasks. Addresses the fundamental problem that token importance is non-monotonic during multi-step reasoning. Reduces peak token usage by 65% for deep-research agents.
S5

Scheduling & Routing

Traditional LLM request scheduling treats each inference call independently. Agentic workloads require program-level and workflow-level scheduling that understands multi-step dependencies, exploits cache locality, and adapts to dynamic execution graphs.

Autellix — Program-level DAG Scheduling
arXiv'252502.13965 Agentic ✓PD —Distributed
First serving system to treat agent programs as first-class citizens (UC Berkeley & Google DeepMind). Formalizes programs as dynamic DAGs, introducing PLAS scheduling (prioritizes programs with lower cumulative service time) and ATLAS (multi-threaded version considering critical paths). Locality-aware load balancer exploits KV cache co-location, 4–15x throughput.
Cortex — Stage-isolated Resource Pooling
arXiv'252510.14126 Agentic ✓PD —Distributed
Stage isolation -- provisioning independent resource pools for each workflow stage (e.g., SQL generation, error correction, execution), each with its own scheduler, cache, and scaling policy. Orchestrator tracks per-request SLO slack.
Murakkab — Declarative Workflow Optimization
arXiv'252508.18298 Agentic ✓PD —Distributed
Microsoft Research proposes declarative abstractions decoupling workflow specification from execution configuration. Profile-guided optimizer and adaptive runtime jointly manage model selection and hardware mapping, 2.8x GPU usage reduction vs LangGraph.
Agent.xpu — On-device SoC Scheduling
arXiv'252506.24045 Agentic ✓PD Disagg
First on-device agentic engine for heterogeneous SoCs (CPU, iGPU, NPU). Assigns prefill to NPU, decode to iGPU. 4.6x lower latency (reactive), 1.6–6.8x higher throughput (proactive). Intel Core Ultra platform.
Preble — Cluster-level KV-aware Scheduling
ICLR'252407.00023repo Agentic —PD —Distributed
Extends prefix caching to cluster level, with E2 scheduling algorithm that co-optimizes KV reuse and load balancing. Global scheduler's prompt-aware load metrics account for eviction costs and sharing potential, 1.5–14.5x mean latency, 2–10x P99.
AI Metropolis — Out-of-order Multi-Agent
MLSys'252411.03519 Agentic ✓PD —
Applies out-of-order execution concepts to multi-agent simulations, dynamically tracking inter-agent dependencies to break conservative synchronization barriers. 1.3–4.15x speedup.
Helium — Query-plan Agentic Workflows
arXiv'262603.16104 Agentic ✓PD —
Models agentic workflows as database query plans with LLMs as first-class operators, combined with template-aware radix-tree scheduling.
Sherlock — Reliable Agentic Workflow
arXiv'252511.00330 Agentic ✓PD —
Combines selective verification with speculative execution for reliable agentic serving. Reduces execution time by 48.7%.
Aragog — Just-in-Time Model Routing
arXiv'252511.20975 Agentic ✓PD —
Adapts per-input at execution time (rather than per-workflow) for Just-in-Time model routing to scale agentic workflows.
SOLA — State-aware SLO Scheduling
MLSys'25 Agentic —PD —
State-aware SLO scheduling for LLM serving. MLSys'25.
SuperServe — Fine-grained Serving
NSDI'252312.16733 Agentic —PD —Distributed
Fine-grained inference serving for unpredictable workloads. NSDI'25.
S6

Learning-based Cache Management

Optimal KV cache TTL and eviction is fundamentally an exploration-exploitation problem: systems must balance exploiting known access patterns against exploring the value of retaining uncertain cache entries. The rich literature on learned caching provides theoretical foundations.

LeCaR — Regret-based LRU/LFU Weighting
HotStorage'18repo Agentic —PD —
Introduces regret-minimization to dynamically weight LRU and LFU. On each cache miss, selects policy based on cumulative regret. Outperforms ARC by >18x when cache is small relative to the working set. Foundation for CACHEUS.
CACHEUS — Fully Adaptive Cache Replacement
FAST'21repo Agentic —PD —
Extends LeCaR to fully adaptive, eliminating all static hyperparameters via gradient-based stochastic hill climbing. Introduces workload classification and new SR-LRU/CR-LFU experts. Evaluated across 17,766 simulations on 329 workloads.
RLCache — Multi-task RL Cache Management
arXiv'191909.13839repo Agentic —PD —
Models cache management as a multi-task RL problem with three agents: caching decision, TTL estimation, and eviction. The TTL estimation agent learns optimal time-to-live -- directly relevant to agentic semantic caching.
Cold-RL — Offline RL for NGINX Eviction
arXiv'252508.12485repo Agentic —PD —
First production-grade RL eviction algorithm, integrated into NGINX. Dueling DQN operates within a strict 500us timeout via ONNX sidecar. Hit rate improves from 0.14 to 0.35 (146% increase), with CPU overhead <2%.
KV Policy — Per-head RL KV Eviction
arXiv'262602.10238 Agentic —PD —
Reformulates KV cache eviction as an RL ranking problem, training independent lightweight agents per attention head. Per-head specialization adapts to different attention patterns -- some heads track recency, others track entity references.
Semantic Cache Bandit — MAB for Cache Eviction
arXiv'252508.07675 Agentic —PD —
First principled framework for semantic cache eviction under unknown query and cost distributions. Three settings: oracle (Reverse Greedy), offline (CUCB-SC), and online adaptive (CLCB-SC-LS). Directly connects bandit theory to LLM serving cache management.
Attention-Gate — In-context KV Eviction
arXiv'242410.12876repo Agentic —PD —
Injects a lightweight module for dynamic per-head KV eviction.
NACL — General KV Cache Eviction
ACL'242408.03675repo Agentic —PD —
General KV cache eviction framework using proxy-token scoring with head-wise random eviction.
LookaheadKV — Future Attention Prediction
arXiv'262603.10899repo Agentic —PD —
Uses learnable special tokens to predict future attention patterns and guide KV eviction. Samsung Labs.
Ada-KV — Adaptive Per-head Budget
NeurIPS'252407.11550repo Agentic —PD —
Adaptive per-head budget allocation for KV cache eviction. NeurIPS'25.
S7

Adjacent Area Optimizations

This section covers four adjacent research directions complementary to agentic infrastructure: chunked prefill (an alternative to PD disaggregation), model-level KV cache compression, speculative decoding and tool calling, and serverless and programmable serving.

Sarathi-Serve — Chunked Prefills
OSDI'242308.16369repo Agentic —PD —
Introduces chunked prefill, splitting long prompts into equal-size chunks and piggybacking decode requests onto prefill batches. Stall-free scheduling achieves 2.6–5.6x serving capacity. Represents a fundamental trade-off against PD disaggregation: chunked prefill mixes phases on one GPU (better memory locality, no KV transfer), while disaggregation separates phases (better phase isolation, KV transfer overhead).
DeepSpeed-FastGen — Dynamic SplitFuse
arXiv'242401.08671repo Agentic —PD —
Microsoft independently proposes similar Dynamic SplitFuse chunking. Presents the same fundamental trade-off against PD disaggregation as Sarathi.
Speculative Tool Calling
arXiv'252512.15834 Agentic ✓PD —
Overlaps tool execution with generation via client-side speculative instances and engine-side tool cache. LLNL.
Speculative Actions — Lossless Agent Speedup
arXiv'252510.04371 Agentic ✓PD —
Generalizes speculation to entire agentic environments -- speculating LLM calls, tool APIs, MCP servers, and human responses, with lossless, as-if-sequential semantics.
ServerlessLLM — Fast Cold-start
OSDI'242401.14351repo Agentic —PD —Distributed
Achieves 10–200x cold-start latency reduction through multi-tier checkpoint loading, live migration, and startup-optimized scheduling. Critical for multi-agent architectures with bursty per-model demand.
FlashInfer — Customizable Attention Engine
MLSys'252501.01005repo★ Best Paper Agentic —PD —
Foundational attention engine: block-sparse KV cache format, JIT-compiled customizable templates, load-balanced scheduling. 29–69% inter-token-latency reduction. Integrated into SGLang, vLLM, MLC-Engine. MLSys'25 Best Paper.
NEO — CPU Offloading for KV + Attention
MLSys'252411.01142repo Agentic —PD —
Offloads KV cache and attention computation to CPU to save GPU memory. MLSys'25.
HCache — Fast KV State Restoration
EuroSys'252410.05004 Agentic —PD —
Restores KV state from intermediate activations, 1.93x TTFT reduction.
Pensieve — Stateful Multi-turn Serving
EuroSys'252312.05516 Agentic ~PD —
Multi-tier KV cache for stateful multi-turn LLM serving.
Helix — Heterogeneous GPU Max-Flow
ASPLOS'252406.01566repo Agentic —PD —Distributed
Max-flow optimization for heterogeneous GPU clusters. ASPLOS'25.
Weaver — Multi-LLM Attention Offloading
ATC'25 Agentic —PD —
Multi-LLM serving with attention offloading.
